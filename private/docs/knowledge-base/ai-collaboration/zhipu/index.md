#### 提示词1：动态规划入门
```
【角色】你是一位ACM竞赛金牌教练，擅长用严谨的数学推导讲解算法。

【背景】学生正在学习GESP五级内容，第一次接触动态规划（DP），需要建立扎实的理论基础。

【任务】请系统讲解"动态规划"的核心思想，以"爬楼梯"问题为例。

【讲解要求】

1. **问题定义**（数学化）
   - 给出爬楼梯问题的形式化定义
   - 明确：n阶楼梯，每次可爬1或2阶，求爬到第n阶的方法数

2. **最优子结构证明**（严谨推导）
   - 证明问题的最优子结构性质
   - 说明：f(n) = f(n-1) + f(n-2) 的数学依据

3. **状态定义**
   - 定义dp[i]的精确含义
   - 说明状态维度的选择理由

4. **状态转移方程**
   - 给出完整的转移方程
   - 证明其正确性

5. **边界条件**
   - 明确初始状态
   - 解释边界条件的设置依据

6. **代码实现**（C++）
   - 提供标准C++实现
   - 代码需包含详细注释
   - 使用规范的命名和格式

7. **复杂度分析**
   - 时间复杂度：O(n)
   - 空间复杂度：O(n)，并说明可优化至O(1)

8. **与递归的对比**
   - 对比记忆化搜索和递推两种实现
   - 分析各自的优缺点

【输出格式】
- 使用清晰的学术层次结构
- 关键公式用 LaTeX 格式表示（如 $f(n) = f(n-1) + f(n-2)$）
- 代码使用规范的C++风格（Google C++ Style）
- 最后给出3道变式练习题（难度递增）
```

**预期输出特点**:
- 完整的数学推导过程
- 严谨的算法正确性证明
- 规范的C++代码（带注释）
- 复杂度分析公式

### 场景2：代码规范与风格指导

#### 提示词3：代码Review示范
```
【角色】你是一位严格的代码审查专家，遵循Google C++ Style Guide。

【背景】GESP三级学生提交了以下代码，需要指出问题并给出改进建议。

【学生代码】
```cpp
#include<iostream>
using namespace std;
int main(){
int n,i,sum=0;
cin>>n;
for(i=1;i<=n;i++){
sum=sum+i;
}
cout<<sum<<endl;
return 0;
}
```

【任务】请进行完整的代码Review。

【Review维度】

1. **格式规范**（逐行检查）
   - 缩进是否统一（应使用2个空格）
   - 括号位置是否符合规范
   - 运算符两侧是否有空格
   - 每行是否超过80字符

2. **命名规范**
   - 变量名是否有意义（n, i, sum 是否足够清晰？）
   - 是否使用驼峰命名或下划线命名
   - 常量是否使用全大写

3. **代码结构**
   - 是否缺少必要的注释
   - 复杂逻辑是否需要提取函数
   - 变量的作用域是否合理

4. **潜在问题**
   - 是否有整数溢出风险（n很大时sum会溢出）
   - 输入是否验证
   - 边界条件是否处理

5. **改进后代码**
   - 给出符合规范的完整代码
   - 标注每处改进

6. **评分**
   - 原代码：/100分
   - 改进后：/100分
   - 主要扣分项说明

【输出格式】
- 使用代码对比格式展示改进
- 每个问题标注严重程度（Error/Warning/Suggestion）
- 语气专业但鼓励，适合12岁学生理解
```

【背景】GESP二级学生遇到编译错误，需要帮助理解错误信息。

```cpp
#include <iostream>

int main() {
    int a, b;
    cin >> a >> b;

        cout << "Max is: " << b << endl;

```

【错误信息】

【任务】帮助学生理解并修复错误。

【讲解步骤】

2. **错误原因**
   - 为什么会出现这个错误？

3. **修复方法**
   - 给出修复后的代码
   - 标注修改处

4. **预防措施**
   - 如何避免此类错误？
   - 给出一个记忆口诀或检查清单

5. **扩展知识**
   - if-else语句的完整语法结构
   - 花括号的使用规范

【语气要求】
- 先肯定学生的整体思路正确
- 将错误视为学习机会
- 使用"我们可以改进"而非"你错了"
```

#### 提示词5：运行时错误调试
```
【角色】你是一位调试专家，擅长运行时错误分析。

【背景】GESP四级学生的代码能编译通过，但运行时结果错误。

【学生代码】
```cpp
#include <iostream>
using namespace std;

int main() {
    int n = 5;
    int arr[n];  // 想创建一个大小为5的数组
    for (int i = 0; i <= n; i++) {  // 初始化数组
        arr[i] = i;
    }
    for (int i = 0; i < n; i++) {
        cout << arr[i] << " ";
    }
    return 0;
}
```

【现象】运行后输出异常，有时程序崩溃。

【任务】进行运行时错误分析。

【分析步骤】

1. **问题诊断**
   - 指出数组越界访问
   - 解释循环条件 i <= n 的问题

2. **内存图解**
   - 用ASCII图示说明数组内存布局
   - 标出越界访问的位置

3. **修复方案**
   - 给出正确的代码
   - 解释为什么 i < n 是正确的

4. **调试技巧**
   - 如何在代码中添加调试输出
   - 如何一步一步跟踪变量值

5. **类似错误防范**
   - 常见的数组越界场景
   - 编程时的检查清单
```

### 场景4：算法思路引导（不直接给答案）

#### 提示词6：竞赛题思路点拨
```
【角色】你是一位竞赛教练，擅长启发式教学，不直接给出答案。

【背景】GESP六级学生正在准备CSP-J，遇到一道较难的题目。

【题目】
给定一个长度为n的数组，找出其中和最大的连续子数组。
示例：[-2,1,-3,4,-1,2,1,-5,4] → 最大和为6（子数组[4,-1,2,1]）

【学生困惑】
学生只会暴力枚举所有子数组，时间复杂度O(n³)，会超时。

【任务】引导学生找到更优解法，但不要直接给出代码。

【引导步骤】

1. **分析暴力解法**
   - 先肯定学生的思路可行
   - 分析时间复杂度：枚举起点O(n) × 枚举终点O(n) × 求和O(n) = O(n³)

2. **第一个提示：前缀和优化**
   - 问：能否将求和优化到O(1)？
   - 引导：前缀和数组 pre[i] = arr[0] + ... + arr[i-1]
   - 让学生自己推导：子数组[i,j]的和 = pre[j+1] - pre[i]
   - 优化后复杂度：O(n²)

3. **第二个提示：分治法**
   - 问：如果不用枚举，能否用分治？
   - 引导：最大子数组和可能在左半、右半，或跨越中点
   - 让学生思考跨越中点的情况如何处理
   - 复杂度：O(n log n)

4. **第三个提示：动态规划（核心突破）**
   - 问：能否一遍扫描就得到答案？
   - 关键问题：以第i个元素结尾的最大子数组和是多少？
   - 引导定义状态：dp[i] = 以arr[i]结尾的最大子数组和
   - 让学生自己推导转移方程

5. **总结提升路径**
   - O(n³) → O(n²) → O(n log n) → O(n)
   - 每种方法的核心思想
   - 鼓励学生先实现O(n)解法

【约束】
- 不要给出完整代码
- 每个提示后留出思考空间
- 最后给出3道类似练习题
```

## 七、使用技巧与注意事项

### 7.1 获得最佳效果的技巧

1. **强调严谨性**
   - 智谱清言擅长学术风格，明确要求"严谨推导"
   - 会给出完整的数学证明和复杂度分析

2. **要求多解法**
   - 明确说"从暴力到最优的完整解法链"
   - 会获得3-4种解法对比

3. **代码规范检查**
   - 智谱清言对代码风格敏感
   - 可要求"符合Google C++ Style"

### 7.2 避免的误区

❌ **避免**："这道题怎么做？"（太笼统）
✅ **推荐**："请用动态规划讲解这道题，给出状态定义、转移方程和C++实现"

❌ **避免**："代码有问题吗？"（没有上下文）
✅ **推荐**："这段GESP四级代码在n=100000时会超时，请分析复杂度并给出优化方案"

❌ **避免**："解释一下快速排序"（太简单）
✅ **推荐**："请对比快速排序和归并排序，分析各自适用场景，给出最优实现"
