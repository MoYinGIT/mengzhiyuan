# 错题本模板

> 标准化错题记录与分析，从错误中学习，避免重复犯错

---

## 为什么需要错题本？

### 错题本的价值

**发现盲点**：
- 记录知识薄弱环节
- 追踪常犯错误类型
- 定位概念理解偏差

**避免重复**：
- 同类错误只犯一次
- 考前重点复习对象
- 针对性强化训练

**进步可视化**：
- 看到错误逐渐减少
- 建立学习信心
- 调整学习策略

## 错误类型分类

### 语法错误（Syntax Error）

**常见表现**：编译不通过

**子类型**：
| 子类型 | 示例 | 预防措施 |
|:---|:---|:---|
| 分号遗漏 | `int a = 5` | 每行结束检查 |
| 括号不匹配 | `if (a > b` | 成对输入 |
| 变量未声明 | `cout << x` | 先声明后使用 |
| 头文件缺失 | 忘记`#include` | 建立代码模板 |

### 逻辑错误（Logic Error）

**常见表现**：能通过编译但结果不对

**子类型**：
| 子类型 | 示例 | 预防措施 |
|:---|:---|:---|
| 边界错误 | `i <= n`应为`i < n` | 画图验证 |
| 初始化错误 | 累加器未清零 | 强制初始化 |
| 条件错误 | `>`写成`<` | 代入测试 |
| 算法错误 | 思路根本错误 | 多举例子 |

### 运行时错误（Runtime Error）

**常见表现**：程序崩溃

**子类型**：
| 子类型 | 示例 | 预防措施 |
|:---|:---|:---|
| 数组越界 | 访问`arr[100]` | 检查下标范围 |
| 除以零 | `a / b`时b=0 | 判断除数非零 |
| 栈溢出 | 递归太深 | 改迭代或剪枝 |
| 内存溢出 | 数组太大 | 估算内存使用 |

### 答案错误（Wrong Answer）

**常见表现**：部分测试点通过

**子类型**：
| 子类型 | 示例 | 预防措施 |
|:---|:---|:---|
| 输出格式 | 多空格/少换行 | 对照题目要求 |
| 特殊情况 | 没处理n=0 | 测试边界值 |
| 精度问题 | 浮点比较 | 使用EPS |
| 数据溢出 | int溢出 | 换long long |

## 错题本示例

### 示例1：语法错误

```markdown
## 错题 #1

- **记录日期**: 2026-03-27
- **题目来源**: 洛谷 - P5703
- **题目名称**: 苹果采购
- **难度级别**: 入门
- **错误类型**: 语法错误 - 分号遗漏
- **涉及知识点**: 基础语法

### 题目描述

输入苹果单价和数量，输出总价。

### 原始错误代码

```cpp
#include <iostream>
using namespace std

int main() {
    int a, b
    cin >> a >> b;
    cout << a * b;
    return 0;
}
```

### 错误信息/结果

编译错误: expected ';' after namespace

### 错误原因分析

**根本原因**: 粗心，忘记加分号

**知识盲点**: 无，纯粗心

**思维误区**: 以为和Python一样不需要分号

### 正确解法

```cpp
#include <iostream>
using namespace std;  // 加分号

int main() {
    int a, b;  // 加分号
    cin >> a >> b;
    cout << a * b;
    return 0;
}
```

### 关键要点

- C++每行语句结束必须有分号
- 结构体/函数定义结束也有分号
- namespace、宏定义、头文件无分号

### 预防措施

- [ ] 写完一行立即检查分号
- [ ] 编译出错首先检查分号
- [ ] 使用IDE自动补全

### 同类题目推荐

- P5701 深基2.例1 简单排序
- P5702 深基2.例2 数字反转
```

### 示例2：逻辑错误

```markdown
## 错题 #2

- **记录日期**: 2026-03-27
- **题目来源**: 洛谷 - P5710
- **题目名称**: 数的性质
- **难度级别**: 入门
- **错误类型**: 逻辑错误 - 边界处理
- **涉及知识点**: 条件判断

### 题目描述

判断一个数是否同时是2和3的倍数。

### 原始错误代码

```cpp
#include <iostream>
using namespace std;

int main() {
    int x;
    cin >> x;
    if (x % 2 == 0 && x % 3 == 0) {
        cout << "YES";
    } else {
        cout << "NO";
    }
    return 0;
}
```

### 错误信息/结果

答案错误: 3/10

### 错误原因分析

**根本原因**: 没仔细看题目要求

**知识盲点**: 题目要求"或"不是"且"

**思维误区**: 想当然认为是同时满足

### 正确解法

```cpp
#include <iostream>
using namespace std;

int main() {
    int x;
    cin >> x;
    if (x % 2 == 0 || x % 3 == 0) {  // 改成 ||
        cout << "YES";
    } else {
        cout << "NO";
    }
    return 0;
}
```

### 关键要点

- && 表示"且"（两个都满足）
- || 表示"或"（满足一个即可）
- 仔细阅读题目，不要想当然

### 预防措施

- [ ] 读题时圈出关键词
- [ ] 用测试数据验证理解
- [ ] 多问自己：题目的意思是？

### 同类题目推荐

- P5711 判断闰年
- P5712 数字判断
```

### 示例3：数组越界

```markdown
## 错题 #3

- **记录日期**: 2026-03-27
- **题目来源**: 洛谷 - P1428
- **题目名称**: 小鱼比可爱
- **难度级别**: 普及-
- **错误类型**: 运行时错误 - 数组越界
- **涉及知识点**: 数组、循环

### 题目描述

N条鱼，每条鱼判断前面有多少条比它矮的。

### 原始错误代码

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, a[105];
    cin >> n;
    for (int i = 1; i <= n; i++) {  // 从1开始
        cin >> a[i];
    }
    for (int i = 1; i <= n; i++) {
        int cnt = 0;
        for (int j = 1; j < i; j++) {
            if (a[j] < a[i]) cnt++;
        }
        cout << cnt << " ";
    }
    return 0;
}
```

### 错误信息/结果

运行时错误: RE (段错误)

### 错误原因分析

**根本原因**: 数组开小了，n=100时a[100]越界

**知识盲点**: 数组大小应该比最大需求大

**思维误区**: 以为a[105]可以存105个元素（0-104）

### 正确解法

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, a[110];  // 开大点，110
    cin >> n;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
    }
    for (int i = 1; i <= n; i++) {
        int cnt = 0;
        for (int j = 1; j < i; j++) {
            if (a[j] < a[i]) cnt++;
        }
        cout << cnt << " ";
    }
    return 0;
}
```

### 关键要点

- 数组下标从0开始
- a[105]的有效下标是0-104
- 留余量：题目说100就开110

### 预防措施

- [ ] 画图确认数组下标范围
- [ ] 数组大小 = 最大需求 + 10
- [ ] 检查循环边界

### 同类题目推荐

- P5727 求极差
- P5728 旗鼓相当的对手
```

## 错题本使用建议

### 记录频率

**每道题都要记录吗？**
- 简单粗心错误：简要记录
- 知识点盲区：详细记录
- 多次犯错：重点标记

**建议比例**：做10道题，记录2-3道典型错题

### 复习计划

**时间安排**：
| 时间节点 | 复习内容 | 方式 |
|:---|:---|:---|
| 当天晚上 | 当天错题 | 重新做一遍 |
| 周末 | 本周错题 | 快速回顾 |
| 考前 | 所有错题 | 重点看标记的 |

### 数字化管理

**使用Kimi辅助**：
```
输入：请帮我分析这道错题的错误类型和知识点
      [粘贴错题本记录]

输入：请根据我的错题本，推荐同类练习题
      [粘贴多个错题]
```

**分类标签**：
- #语法错误
- #逻辑错误
- #算法错误
- #粗心错误
- #待复习
