## 📖 精选内容与解读

### 总叙

> **弟子规，圣人训。首孝悌，次谨信。**

**意思**：《弟子规》是圣人的教诲，首先要孝顺父母、友爱兄弟，其次要谨慎守信。

**编程连接**：
```
写程序也要有"规矩"：
- 代码要规范（首孝悌）
- 变量命名要谨慎（次谨信）
- 注释要守信（说到做到，注释要写清楚）
```

---

### 入则孝 - 学习的态度

> **父母呼，应勿缓。父母命，行勿懒。**

**意思**：父母叫唤，要及时应答；父母吩咐，要立即行动，不偷懒。

**编程版的"孝"**：
```
"父母"换成"学习目标"

学习目标呼，应勿缓：
- 定好的学习计划要及时执行
- 不要拖延

学习目标命，行勿懒：
- 不要偷懒
- 坚持每天练习
```

**Scratch 习惯打卡器**：
```scratch
定义 检查习惯
    如果 今天练习了编程 那么
        说 "好习惯！+1分" 持续 2 秒
        将 连续天数 增加 1
        如果 连续天数 > 最高记录 那么
            说 "创纪录了！" 持续 2 秒
            将 最高记录 设为 连续天数
        结束如果
    否则
        说 "今天还没练习哦" 持续 2 秒
        将 连续天数 设为 0
    结束如果
结束定义
```

> **出必告，反必面。居有常，业无变。**

**意思**：出门要告诉父母，回来要报平安。起居有规律，事业不随便改变。

**编程习惯**：
```
出必告 = 开始写代码前，想清楚要做什么
反必面 = 写完代码，要测试验证

居有常 = 代码要有固定格式
业无变 = 不要随意改变已经验证好的逻辑
```

### 谨 - 谨慎细致

> **朝起早，夜眠迟。老易至，惜此时。**

**意思**：早上早起，晚上晚睡（但不要熬夜）。年纪很快就老了，要珍惜时间。

**给小朋友的话**：
> 写程序也要珍惜时间：
- 不要一边写一边玩
- 专注 30 分钟，比分散 1 小时效果好
- 休息也很重要，但不要拖延

**时间管理小技巧**：
```
番茄工作法：
🍅 专注 25 分钟
☕ 休息 5 分钟
🍅 再来一个 25 分钟

用 Scratch 做个番茄钟：
- 倒计时 25 分钟
- 时间到播放提醒音
- 记录完成了几个番茄
```

---

> **事勿忙，忙多错。勿畏难，勿轻略。**

**意思**：做事不要匆忙，匆忙容易出错。不要怕困难，也不要轻视（认为简单就马虎）。

**编程黄金法则**：
```
事勿忙，忙多错：
- 不要急着写代码
- 先想清楚逻辑
- 画流程图是个好办法

勿畏难：
- Bug 不要怕
- 多难的题都有解法
- 可以查资料、问老师

勿轻略：
- 简单的题也要认真
- 边界情况要考虑
- 测试要全面
```

### 信 - 诚信与坚持

> **凡出言，信为先。诈与妄，奚可焉。**

**意思**：说话，诚信最重要。欺骗和胡说，怎么可以呢？

**编程中的"信"**：
```
对自己的承诺要守信：
- 定好今天要完成的功能，要完成
- 说好了要优化代码，就要做到

对他人的承诺要守信：
- 答应了帮同学 debug，要认真帮
- 说好了分享代码，就要分享

代码要"诚实"：
- 不要抄袭
- 不要假装会
- 不会就是不会，学了就会
```

### 余力学文 - 学习方法

> **不力行，但学文。长浮华，成何人。**

**意思**：不实践，只学理论，会变得浮华，成为什么样的人呢？

**编程学习的关键**：
```
❌ 不要：只看视频不动手
❌ 不要：只看书不写代码
❌ 不要：只听老师讲自己不思考

✅ 要：学了马上练
✅ 要：做项目，学以致用
✅ 要：遇到问题自己先想办法
```

**力行清单**：
- [ ] 每天写至少 30 行代码
- [ ] 每周完成一个小项目
- [ ] 每月分享一个作品

---

> **但力行，不学文。任己见，昧理真。**

**意思**：只实践，不学习理论，会固执己见，不明白真正的道理。

**平衡很重要**：
```
不能只写代码不学习：
- 要学算法原理
- 要看优秀的代码
- 要了解新的技术

理论和实践要结合：
学了排序算法 → 动手实现 → 应用到项目中
```

---

> **读书法，有三到。心眼口，信皆要。**

**意思**：读书的方法，有三到：心到、眼到、口到，都很重要。

**编程学习"三到"**：
```
心到：专心，不分心
眼到：仔细看代码，看文档
手到：动手写，不要只看

再加一个：
脑到：思考为什么这样写
```

---

> **方读此，勿慕彼。此未终，彼勿起。**

**意思**：正在读这本书，不要想着那本书。这本没读完，不要开始另一本。

**学习的大忌**：
```
❌ 不要：
- Scratch 还没学好，就想学 Python
- Python 还没学好，就想学 C++
- 这个教程看一半，又去看那个教程

✅ 要：
- 把一个学扎实
- 学透彻
- 再学下一个
```

## 🌟 弟子规与编程规范

### 代码规范版"弟子规"

```
变量名，要有义。勿aabb，让人迷。
（变量名要有意义，不要用a、b这样无意义的命名）

写代码，要缩进。层级清，易阅读。
（代码要缩进，层级清晰）

加注释，说明白。他人看，不费劲。
（要写注释，让别人能看懂）

函数短，功能单。一长串，难维护。
（函数要短，功能单一）

测边界，莫轻略。错处理，要完善。
（要测试边界情况，错误处理要完善）
```
