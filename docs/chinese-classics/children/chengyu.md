## 📖 精选成语故事

### 1. 亡羊补牢

**故事**：
> 从前有个牧民，羊圈破了，狼进来偷走了一只羊。邻居劝他修羊圈，他说："羊都丢了，还修什么？"第二天，狼又偷走一只羊。他后悔没听劝告，赶紧修好了羊圈，再也没有丢羊。

**意思**：出了问题以后想办法补救，还不算晚。

**编程连接 - Bug 修复**：
```
程序出 Bug 就像羊圈破了：

❌ 错误态度：
"反正已经出错了，算了"
→ 结果：Bug 越来越多

✅ 正确态度（亡羊补牢）：
"虽然出了问题，但我要赶紧修复"
→ 结果：程序恢复正常

记住：发现 Bug 要马上修，不要拖延！
```

**Scratch 项目：Bug 修复模拟器**
```scratch
定义 羊圈
    将 羊圈状态 设为 "破损"
    将 羊数量 设为 10
    说 "羊圈破损，有 10 只羊" 持续 2 秒
结束定义

定义 狼来了
    如果 羊圈状态 = "破损" 那么
        将 羊数量 增加 -1
        说 "狼偷走了一只羊！还剩" 羊数量 "只" 持续 2 秒
    否则
        说 "狼来了，但羊圈完好，羊都安全！" 持续 2 秒
    结束如果
结束定义

定义 修羊圈
    将 羊圈状态 设为 "完好"
    说 "羊圈修好了！" 持续 2 秒
结束定义

当 绿旗被点击
    羊圈
    狼来了  // 第一次，不修补
    狼来了  // 第二次，丢了两只羊
    修羊圈  // 亡羊补牢
    狼来了  // 这次安全了
结束
```

### 2. 守株待兔

**故事**：
> 宋国有个农夫，田里有个树桩。一天，一只兔子撞在树桩上死了。农夫捡回家吃了。他想："要是天天都有兔子撞死就好了！"于是他不再耕田，天天守在树桩旁等兔子。结果再也没有兔子来，田地荒芜了。

**意思**：不主动努力，存侥幸心理，希望得到意外收获。

**编程连接 - 不要指望运气**：
```
❌ 守株待兔式编程：
"我写了个程序，希望它能运行"
→ 不测试、不调试
→ 结果：到处都是 Bug

✅ 正确态度：
"我写了个程序，我要验证它是否正确"
→ 仔细测试
→ 结果：程序稳定可靠

编程不能只靠运气，要主动思考和验证！
```

### 3. 愚公移山

**故事**：
> 北山有个叫愚公的老人，门前有两座大山挡路。他带领家人挖山，要把山移走。邻居智叟笑他傻："你挖得完吗？"愚公说："我死了有儿子，儿子死了有孙子，子子孙孙无穷尽，山不会增高，总有一天能挖平！"天帝被感动，派神仙把山背走了。

**意思**：只要有毅力和决心，再难的事情也能成功。

**编程连接 - 坚持不懈**：
```
学编程就像愚公移山：

大山 = 复杂的算法、难懂的语法
每天挖一点 = 每天学习、练习

愚公说："子子孙孙无穷尽"
我们说："每天进步一点点"

不要因为一道题难就放弃
不要因为一个 Bug 就崩溃
坚持下去，总有一天你会"移平大山"
```

**Scratch 项目：移山挑战**
```scratch
定义 移山 接收 天数
    将 山的大小 设为 100
    重复执行 天数 次
        将 山的大小 增加 -1
        说 连接 "第" 天数 "天，山还剩余：" 山的大小 持续 0.5 秒
    结束重复
    如果 山的大小 <= 0 那么
        说 "山被移平了！你成功了！" 持续 3 秒
    否则
        说 "继续努力，坚持就是胜利！" 持续 2 秒
    结束如果
结束定义

当 绿旗被点击
    询问 "你要坚持多少天？" 并等待
    移山 回答
结束
```

### 4. 画蛇添足

**故事**：
> 楚国有人祭祀后，赏给门客一壶酒。门客们商量："酒少人多，不如比赛画蛇，谁先画完谁喝。"一人先画完，拿起酒壶，却左手拿酒，右手继续画："我还能给蛇画脚！"没等他画完，另一人画完了蛇，抢过酒说："蛇本来没有脚，你画的是什么？"说完把酒喝了。

**意思**：做多余的事，反而不恰当。

**编程连接 - 简洁为美**：
```
好的代码：简单、清晰、够用

❌ 画蛇添足式代码：
- 写了一大堆没必要的功能
- 代码复杂难懂
- 容易出错

✅ 简洁的代码：
- 只做需要做的事
- 代码清晰易读
- 维护简单

记住：大道至简，不要画蛇添足！
```

### 5. 井底之蛙

**故事**：
> 一只青蛙住在井底，它觉得天空只有井口那么大。一只海龟路过，告诉它大海的广阔。青蛙才知道世界有多大。

**意思**：眼界狭窄，见识短浅。

**编程连接 - 开阔视野**：
```
不要只做"井底之蛙"：

❌ 局限：
- 只学一种编程语言
- 只看一个教程
- 只做一个类型的项目

✅ 开阔：
- 多学几种语言
- 多看不同的教程
- 尝试各种类型的项目
- 和别人交流，学习他人的代码

世界很大，技术很多，要保持好奇心！
```

### 6. 一鸣惊人

**故事**：
> 楚庄王继位三年，不理朝政，整天玩乐。大臣们很担心。有个大臣用隐语问他："山上有只鸟，三年不飞不鸣，是什么鸟？"楚庄王说："三年不飞，一飞冲天；三年不鸣，一鸣惊人。"后来楚庄王果然励精图治，成为春秋五霸之一。

**意思**：平时不露声色，突然做出惊人的成绩。

**编程连接 - 厚积薄发**：
```
学习编程要"厚积薄发"：

三年不飞 = 默默学习、积累
一飞冲天 = 做出优秀作品

不要急于求成
先打好基础
积累够了
自然能"一鸣惊人"
```

## 🎮 成语猜猜乐

```scratch
定义 初始化成语库

    将 解释库 设为 [

        不主动努力存侥幸心理,
        有毅力再难的事也能成功,

        眼界狭窄见识短浅,
        平时积累突然做出惊人成绩

结束定义

定义 出题
    将 当前题 设为 随机数从 1 到 成语库 的长度
    说 解释库 的第 当前题 项 持续 4 秒
    说 "这是什么成语？" 持续 2 秒
结束定义

定义 检查答案 接收 答案
    如果 答案 = 成语库 的第 当前题 项 那么
        说 "答对了！太棒了！" 持续 2 秒
        将 分数 增加 20
    否则
        说 连接 "错了，正确答案是：" 成语库 的第 当前题 项 持续 3 秒
    结束如果
结束定义

当 绿旗被点击
    初始化成语库
    将 分数 设为 0
    重复执行 6 次
        出题
        等待 5 秒
    结束重复
    说 连接 "游戏结束！得分：" 分数 持续 3 秒
结束
```
