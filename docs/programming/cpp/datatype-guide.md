## 一、核心概念

### 三要素关系图

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  数据类型    │ ───→ │   内存空间   │ ←─── │   变量名    │
│  (int/float)│      │  (4字节/8字节)│      │  (age/pi)   │
└─────────────┘      └─────────────┘      └─────────────┘
       ↑                      ↑                  ↑
       │                      │                  │
   决定大小               存储实际值          程序员使用
```

### 一句话总结

> **数据类型** 决定占用内存的大小  
> **变量名** 是内存空间的别名（便于程序员使用）  
003e **内存地址** 是变量的「门牌号」（计算机使用）

## 二、常见数据类型大小

| 类型 | 大小 | 说明 | 示例 |
|:---:|:---:|:---|:---|
| `char` | 1 字节 | 存储单个字符 | `'A'`, `'9'` |
| `bool` | 1 字节 | 真/假 | `true`, `false` |
| `short` | 2 字节 | 短整型 | `100`, `-50` |
| `int` | 4 字节 | 整型（最常用）| `42`, `-100` |
| `float` | 4 字节 | 单精度浮点 | `3.14`, `2.5` |
| `double` | 8 字节 | 双精度浮点 | `3.14159` |
| `long long` | 8 字节 | 长长整型 | `10000000000` |

> 💡 **注意**：1 字节 = 8 位（bit），是内存的最小寻址单位

## 三、图解：变量如何存储

### 示例代码

```cpp
int age = 25;
float pi = 3.14;
char grade = 'A';
```

### 内存中的样子

```
内存地址    变量名    类型     占用     存储的值
─────────   ──────   ─────   ──────    ─────────
0x1000      age      int     4字节  →  [25]
0x1004      pi       float   4字节  →  [3.14]
0x1008      grade    char    1字节  →  ['A']
0x1009      (空)                      [空闲]
```

### 关键理解

1. **地址连续**：变量按声明顺序连续存放
2. **类型决定步长**：`int` 占 4 字节，下一个变量从 `0x1004` 开始
3. **char 特殊**：只占用 1 字节，地址只增加 1

## 四、深入理解：为什么需要类型？

### 类比：盒子的标签

想象内存是一个仓库，数据类型是盒子上的标签：

| 类型 | 类比 | 容量 | 用途 |
|:---|:---|:---:|:---|
| `char` | 小首饰盒 | 很小 | 装单个字符 |
| `int` | 标准鞋盒 | 中等 | 装整数 |
| `float` | 鞋盒（精密仪器）| 中等 | 装小数 |
| `double` | 大行李箱 | 大 | 装高精度小数 |

### 没有类型会怎样？

```cpp
// 如果内存只是一堆 0 和 1
00000000 00000000 00000000 00011001  // 这是什么？

// 解释为 int → 25
// 解释为 float → 0.000000  （完全不同的值！）
// 解释为 char[] → '\0\0\0\x19'  （字符串！）
```

> 🔑 **核心**：类型告诉计算机「如何解读」这串 0 和 1

## 五、变量名 vs 内存地址

### 代码视角

```cpp
int age = 25;  // 程序员使用变量名
```

### 计算机视角

```
地址: 0x1000
值:   00000000 00000000 00000000 00011001  (25的二进制)
```

### 关系

```
程序员写：        计算机看：
age    ────────→  0x1000 地址
        编译器
        转换
```

> 💡 **变量名是给程序员用的**，最终都会被转换为内存地址

## 六、sizeof 运算符

### 用法

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "int: " << sizeof(int) << " 字节\n";
    cout << "float: " << sizeof(float) << " 字节\n";
    cout << "double: " << sizeof(double) << " 字节\n";
    cout << "char: " << sizeof(char) << " 字节\n";
    
    int age = 25;
    cout << "age 变量: " << sizeof(age) << " 字节\n";
    
    return 0;
}
```

### 典型输出

```
int: 4 字节
float: 4 字节
double: 8 字节
char: 1 字节
age 变量: 4 字节
```

## 七、常见误区 ⚠️

### ❌ 误区 1：int 和 float 都是 4 字节，可以互换

```cpp
int a = 3.14;    // ❌ 错误！小数部分被截断，a = 3
float b = 1000000000;  // ⚠️ 精度可能丢失
```

**原因**：虽然都是 4 字节，但**存储格式完全不同**
- `int`：直接存储整数的二进制
- `float`：IEEE 754 标准（符号位 + 指数 + 尾数）

---

### ❌ 误区 2：变量名就是值

```cpp
int a = 10;
int b = a;   // 是把 a 的值复制给 b

// 内存中：
// 0x1000: a = 10
// 0x1004: b = 10  （两份独立的数据）
```

> 💡 赋值操作是「复制值」，不是「共享空间」

---

### ❌ 误区 3：char 只能存字母

```cpp
char c = 65;     // ✅ 完全合法！c = 'A'
char d = '9';    // ✅ 字符 '9'，不是数字 9
char e = 9;      // ✅ 这是「换行符」控制字符
```

> 💡 `char` 本质是「1 字节整数」，可以存 -128~127 或 0~255

### 题目 1：sizeof 计算

```cpp
char c = 'A';
int i = 100;
double d = 3.14;
```

问：这三个变量总共占用多少内存？

<details>
<summary>点击查看答案</summary>

- `char`: 1 字节
- `int`: 4 字节
- `double`: 8 字节

**总共：13 字节**

</details>

---

### 题目 2：类型转换

```cpp
int a = 3.7;      // a = ?
float b = 5 / 2;  // b = ?
float c = 5.0 / 2; // c = ?
```

<details>
<summary>点击查看答案</summary>

- `a = 3` （小数部分截断）
- `b = 2` （整数除法，结果再转 float）
- `c = 2.5` （浮点数除法）

</details>

---

### 题目 3：内存布局

```cpp
char a = 'X';
int b = 10;
char c = 'Y';
```

假设 `a` 在地址 `0x1000`，问 `c` 的地址是多少？

<details>
<summary>点击查看答案</summary>

- `a`: 0x1000 (char, 1字节)
- `b`: 0x1001 (int, 4字节)
- `c`: 0x1005 (char, 1字节)

**c 的地址是 0x1005**

</details>
