## 📚 知识模块

### 模块一：计算机基础与Scratch入门

#### 1.1 认识计算机

**什么是计算机？**

计算机由两部分组成：
- **硬件**（Hardware）：看得见摸得着的部分
  - 显示器、键盘、鼠标
  - 主机（CPU、内存、硬盘）
  
- **软件**（Software）：运行在计算机上的程序
  - 操作系统（Windows、MacOS）
  - 应用程序（微信、浏览器、Scratch）

**什么是编程？**

编程就是告诉计算机「做什么」和「怎么做」。

就像给小狗下指令：
- "坐下" → 小狗坐下
- "握手" → 小狗伸爪子

编程语言就是我们和计算机交流的语言。

#### 1.2 Scratch 编程平台

**Scratch 是什么？**

Scratch 是麻省理工学院（MIT）开发的图形化编程工具，就像搭积木一样编程！

**界面介绍**

Scratch 界面分为五大区域：

![Scratch 界面示意图](/scratch-interface.png)

| 区域 | 位置 | 功能 |
|:---|:---|:---|
| **模块区** | 左侧 | 各种编程积木分类存放（运动、外观、声音、事件、控制、侦测、运算、变量）|
| **舞台区** | 中间上方 | 程序运行后显示效果的地方 |
| **角色区** | 右侧上方 | 管理和选择要编程的角色 |
| **背景区** | 右侧下方 | 管理和切换场景背景 |
| **脚本区** | 中间下方 | 拖拽积木编写程序的地方 |

| 区域 | 功能 |
|:---|:---|
| **舞台区** | 程序运行后显示效果的地方 |
| **角色区** | 管理和选择要编程的角色 |

| **模块区** | 各种编程积木分类存放 |

**第一个程序：让小猫打招呼**

```scratch
当绿旗被点击
说"你好，世界！"2秒
```

**操作步骤**：
1. 打开 Scratch（[在线版](https://scratch.mit.edu/create)）
2. 从「事件」模块拖出「当绿旗被点击」
3. 从「外观」模块拖出「说...2秒」
4. 修改文字为「你好，世界！」
5. 点击舞台区上方的绿旗运行

🎉 恭喜你，完成了第一个程序！

### 模块二：编程数学基础

#### 2.1 坐标系

**什么是坐标？**

坐标就像电影院座位号，用两个数字确定一个位置：
- **排号**（x轴）：左右位置
- **座位号**（y轴）：前后位置

**Scratch 坐标系**

Scratch 舞台使用直角坐标系，中心点为原点 (0, 0)：

| 位置 | x 坐标 | y 坐标 |
|:---|:---:|:---:|
| 中心点 | 0 | 0 |
| 最左边 | -240 | 任意 |
| 最右边 | 240 | 任意 |
| 最上边 | 任意 | 180 |
| 最下边 | 任意 | -180 |

**舞台大小**：宽 480（-240 到 240），高 360（-180 到 180）

**角度方向**：
- 0° = 向上
- 90° = 向右（默认方向）
- 180° = 向下
- -90° = 向左

**常用坐标积木**

| 积木 | 功能 |
|:---|:---|
| `移到 x: (0) y: (0)` | 瞬间移动到指定位置 |
| `在 (1) 秒内滑行到 x: (0) y: (0)` | 平滑移动到指定位置 |
| `x坐标` | 获取当前x坐标值 |
| `y坐标` | 获取当前y坐标值 |

**练习：让小猫走到舞台中央**

```scratch
当绿旗被点击
移到 x: (0) y: (0)
说"我到中心啦！"2秒
```

#### 2.2 角度与方向

**角度方向**

角度表示角色的旋转方向，一圈 360°：

| 角度 | 方向 |
|:---:|:---|
| 0° | 向上 |
| 90° | 向右（默认）|
| 180° | 向下 |
| -90° 或 270° | 向左 |

**常用方向积木**

| 积木 | 功能 |
|:---|:---|
| `面向 (90) 方向` | 设置面向的角度 |
| `右转 (15) 度` | 向右旋转 |
| `左转 (15) 度` | 向左旋转 |
| `面向鼠标指针` | 始终面向鼠标 |

**练习：小猫转圈**

```scratch
当绿旗被点击
重复执行 (12) 次
    移动 (30) 步
    右转 (30) 度
结束
```

#### 2.3 随机数

**什么是随机数？**

随机数就是不确定的数，就像掷骰子。

**随机数积木**

```scratch
在 (1) 和 (10) 之间取随机数
```

用途：
- 随机位置：`x: 随机数(-240, 240)`
- 随机颜色：配合「将颜色特效增加」
- 游戏随机性：敌人随机出现

**练习：小猫随机移动**

```scratch
当绿旗被点击
重复执行
    在 (1) 秒内滑行到 x: (随机数 (-240) 到 (240)) y: (随机数 (-180) 到 (180))
    等待 (0.5) 秒
结束
```

### 模块三：三大基本结构

#### 3.1 顺序结构

**什么是顺序结构？**

程序从上到下，依次执行。

就像做早餐：
1. 起床
2. 刷牙
3. 做早餐
4. 吃早餐

**Scratch 示例**

```scratch
当绿旗被点击
说"第一步"2秒      ← 先执行
说"第二步"2秒      ← 再执行
说"第三步"2秒      ← 最后执行
```

#### 3.2 分支结构（如果...那么...）

**什么是分支结构？**

根据条件决定执行哪段代码。

就像出门：
- 如果下雨 → 带伞
- 如果不下雨 → 不带伞

**Scratch 积木**

```scratch
如果 <> 那么
    // 条件为真时执行
结束

如果 <> 那么
    // 条件为真时执行
否则
    // 条件为假时执行
结束
```

**常用条件**

| 条件积木 | 含义 |
|:---|:---|
| `碰到鼠标指针？` | 是否碰到鼠标 |
| `碰到颜色 (#FF0000)？` | 是否碰到红色 |
| `按下空格键？` | 是否按了空格 |
| `x坐标 > 0` | x是否在右边 |
| `< () > ()` | 比较两个数 |

**练习：点击小猫才说话**

```scratch
当角色被点击
说"你点到我啦！"2秒
```

**练习：小猫在左边还是右边？**

```scratch
当绿旗被点击
重复执行
    如果 <(x坐标) > (0)> 那么
        说"我在右边"0.5秒
    否则
        说"我在左边"0.5秒
    结束
    移动 (10) 步
    如果 <碰到边缘？> 那么
        碰到边缘就反弹
    结束
结束
```

#### 3.3 循环结构

**什么是循环结构？**

重复执行一段代码。

就像跳绳：跳一下、再跳一下...直到累了。

**三种循环积木**

```scratch
// 1. 重复执行（无限循环）
重复执行
    // 一直执行
结束

// 2. 重复执行 N 次
重复执行 (10) 次
    // 执行10次
结束

// 3. 重复执行直到（条件循环）
重复执行直到 <条件>
    // 直到条件为真才停止
结束
```

**练习：小猫边走边说数字**

```scratch
当绿旗被点击
重复执行 (5) 次
    移动 (20) 步
    说 (计数器) 2秒  // 计数器从1到5
结束
说"走完了！"2秒
```

### 模块四：角色的操作

#### 4.1 角色基本操作

**导入角色**

1. 点击角色区右下角的「选择角色」
2. 可以从库中选择，或上传自己的图片

**角色属性积木**

| 类别 | 积木 | 功能 |
|:---|:---|:---|
| 外观 | `说 [你好] 2秒` | 气泡说话 |
| 外观 | `思考 [嗯...] 2秒` | 思考气泡 |
| 外观 | `将大小设为 (100)` | 改变大小 |
| 外观 | `显示` / `隐藏` | 显示或隐藏 |
| 外观 | `将颜色特效增加 (25)` | 变色效果 |
| 外观 | `下一个造型` | 切换造型 |
| 运动 | `移动 (10) 步` | 向前移动 |
| 运动 | `右转 (15) 度` | 旋转 |
| 运动 | `面向 (90) 方向` | 设置方向 |
| 运动 | `移到 x: (0) y: (0)` | 设置位置 |
| 运动 | `碰到边缘就反弹` | 边缘反弹 |

#### 4.2 造型与动画

**什么是造型？**

角色的不同姿势，就像翻书动画。

**制作走路动画**

```scratch
当绿旗被点击
重复执行
    下一个造型
    移动 (10) 步
    等待 (0.2) 秒
    如果 <碰到边缘？> 那么
        碰到边缘就反弹
    结束
结束
```

#### 4.3 声音

**播放声音积木**

```scratch
播放声音 [喵] 直到播放完毕
播放声音 [喵]              // 不等待，继续执行
停止所有声音
```

### 模块五：背景与场景

**背景与角色的区别**

| | 角色 | 背景 |
|:---|:---|:---|
| 数量 | 可以有多个 | 同时只能有一个 |
| 移动 | 可以移动 | 不能移动 |
| 代码 | 可以有自己的代码 | 也可以有代码 |

**背景切换**

```scratch
当绿旗被点击
重复执行
    换成 [背景1 v] 背景
    等待 (2) 秒
    换成 [背景2 v] 背景
    等待 (2) 秒
结束
```

### 模块六：事件与交互

#### 6.1 事件积木

事件积木是程序的「起点」，告诉程序什么时候开始运行。

| 积木 | 触发条件 |
|:---|:---|
| `当绿旗被点击` | 点击绿旗 |
| `当角色被点击` | 点击角色 |
| `当按下 [空格 v] 键` | 按下键盘 |
| `当背景换成 [背景1 v]` | 背景切换 |
| `当接收到 [消息1 v]` | 收到广播 |

#### 6.2 键盘控制

**练习：用方向键控制小猫**

```scratch
// 这段代码放到小猫身上

当按下 [向上 v] 箭头
面向 (0) 方向
移动 (10) 步

当按下 [向下 v] 箭头
面向 (180) 方向
移动 (10) 步

当按下 [向左 v] 箭头
面向 (-90) 方向
移动 (10) 步

当按下 [向右 v] 箭头
面向 (90) 方向
移动 (10) 步
```

#### 6.3 广播消息

**什么是广播？**

角色之间「喊话」，一个角色发消息，其他角色接收。

**示例：点击绿旗，小猫和背景同时动起来**

```scratch
// 小猫的代码
当绿旗被点击
广播 [开始游戏 v]

当接收到 [开始游戏 v]
重复执行
    移动 (10) 步
    碰到边缘就反弹
结束

// 背景的代码
当接收到 [开始游戏 v]
重复执行
    将颜色特效增加 (5)
    等待 (0.5) 秒
结束
```

## 🎯 三经典智慧

### 《易经》· 一阴一阳之谓道

编程中的「如果...那么...」就是阴阳：
- 条件为真 → 执行这段
- 条件为假 → 执行那段

就像开关，要么开，要么关，没有中间状态。

### 《道德经》· 千里之行，始于足下

不要觉得编程难，从第一个积木开始，一步一步，终能做出精彩的作品。

### 《论语》· 学而不思则罔

学编程要多动手，光看不练学不会。每学完一个知识点，都要自己做一个小作品巩固。
