## 📚 模块一：列表（List）

### 1.1 什么是列表？

**列表就是可以装很多东西的盒子**，可以存数字、文字，甚至其他列表！

**生活中的列表**：
- 购物清单：苹果、香蕉、牛奶
- 课程表：语文、数学、英语
- 成绩单：85、92、78、90

### 1.2 创建列表

```
点击「变量」→「建立一个列表」

列表名称：购物清单
适用于所有角色 ✓
```

创建后舞台上会显示列表内容，可以勾选隐藏。

### 1.3 列表操作积木

| 积木 | 功能 | 示例 |
|:---|:---|:---|
| `将东西加入购物清单` | 末尾添加 | 添加「苹果」 |
| `删除购物清单的第1项` | 删除指定位置 | 删除第1个 |
| `删除购物清单的全部` | 清空列表 | 重新开始 |
| `购物清单的第1项` | 获取指定项 | 获取第1个 |
| `购物清单` | 整个列表 | 显示所有 |
| `购物清单的长度` | 列表项数 | 有几个？ |
| `购物清单包含苹果？` | 判断是否包含 | 有没有？ |

### 1.4 列表应用案例

**案例一：购物清单**

```scratch
当绿旗被点击
删除购物清单的全部
将苹果加入购物清单
将香蕉加入购物清单
将牛奶加入购物清单
说购物清单 2秒
说购物清单的长度 2秒  // 说"3"
```

**案例二：随机点名**

```scratch
// 创建列表「学生名单」
// 预先填入：小明、小红、小刚、小丽、小强

当绿旗被点击
将计数器设为 1
重复执行 5 次
    将学生名单的第计数器项加入临时名单
    将计数器增加 1
结束

当角色被点击
如果临时名单的长度 > 0 那么
    随机设为 1 到 临时名单的长度 之间的随机数
    说临时名单的第随机项  // 随机点到一个人
    删除临时名单的第随机项  // 去掉已点过的人
否则
    说所有人都点过了！
结束
```

**案例三：成绩统计**

```scratch
// 列表「成绩」已预设

当绿旗被点击
// 计算总分
将总分设为 0
将计数器设为 1
重复执行 成绩的长度 次
    将总分增加 成绩的第计数器项
    将计数器增加 1
结束

// 计算平均分
将平均分设为 总分 / 成绩的长度

// 找出最高分和最低分
将最高分设为 成绩的第1项
将最低分设为 成绩的第1项
将计数器设为 2
重复执行 成绩的长度 - 1 次
    如果 成绩的第计数器项 > 最高分 那么
        将最高分设为 成绩的第计数器项
    结束
    如果 成绩的第计数器项 < 最低分 那么
        将最低分设为 成绩的第计数器项
    结束
    将计数器增加 1
结束

说总分、平均分、最高分、最低分
```

## 📚 模块二：字符串处理

### 2.1 什么是字符串？

**字符串就是文字**，用引号包起来的内容。

```scratch
"Hello"      // 字符串
"你好世界"    // 字符串
"123"        // 这也是字符串，不是数字！
```

### 2.2 字符串操作

Scratch中的字符串积木在「运算」模块：

| 积木 | 功能 |
|:---|:---|
| `连接你好和世界` | 拼接字符串 |
| `你好world的第1个字符` | 获取指定位置字符 |
| `hello的长度` | 字符串长度 |
| `包含世界吗？` | 判断是否包含 |

**案例：文字游戏**

```scratch
当绿旗被点击
询问你叫什么名字？并等待
将名字设为回答

// 拼接欢迎语
说连接你好，和名字 2秒
说连接欢迎你来到编程世界！ 2秒

// 显示名字长度
说连接你的名字有和名字的长度和个字 2秒

// 显示名字首字母
说连接你名字的第一个字是：和名字的第1个字符 2秒
```

### 2.3 字符串遍历

逐个处理字符串中的每个字符：

```scratch
当绿旗被点击
将文字设为 "Hello"
将计数器设为 1
重复执行 文字的长度 次
    说 文字的第计数器个字符
    等待 0.5 秒
    将计数器增加 1
结束
```

## 📚 模块三：自制积木（函数）

### 3.1 为什么要用函数？

**重复的代码，写一次，用多次！**

比如「画正方形」的代码：
```scratch
// 每次画正方形都要写这些
重复执行 4 次
    移动 100 步
    右转 90 度
结束
```

如果能变成：
```scratch
画正方形  // 一行搞定！
```

这就是**函数**（自制积木）的作用！

### 3.2 创建自制积木

**步骤**：
1. 点击「自制积木」→「制作新的积木」
2. 输入积木名称，如「画正方形」
3. 编写积木要执行的代码
4. 完成后像使用普通积木一样使用

```scratch
定义 画正方形
重复执行 4 次
    移动 100 步
    右转 90 度
结束
```

**使用**：
```scratch
当绿旗被点击
画正方形  // 调用自制积木
等待 1 秒
右转 45 度
画正方形  // 再画一个
```

### 3.3 带参数的函数

**问题**：想画不同大小的正方形怎么办？

**解决**：添加参数！

```scratch
定义 画正方形 边长
重复执行 4 次
    移动 边长 步
    右转 90 度
结束
```

**使用**：
```scratch
画正方形 100  // 画100的正方形
画正方形 50   // 画50的正方形
画正方形 200  // 画200的正方形
```

### 3.4 多个参数

```scratch
定义 画长方形 长 宽
重复执行 2 次
    移动 长 步
    右转 90 度
    移动 宽 步
    右转 90 度
结束
```

**使用**：
```scratch
画长方形 100 50   // 长100，宽50
画长方形 200 100  // 长200，宽100
```

### 3.5 带返回值的函数

**问题**：想做一个「计算平均值」的函数，把结果存起来怎么办？

**解决**：使用返回值！

```scratch
定义 计算平均分 总分 人数
将平均分设为 总分 / 人数
```

**注意**：在Scratch中，自制积木不能直接返回值，需要用变量间接实现。

### 3.6 函数应用案例

**案例一：画多边形**

```scratch
定义 画多边形 边数 边长
重复执行 边数 次
    移动 边长 步
    右转 360 / 边数 度
结束
```

**使用**：
```scratch
当绿旗被点击
画多边形 3 100   // 画三角形
等待 1 秒
画多边形 5 100   // 画五边形
等待 1 秒
画多边形 8 100   // 画八边形
```

**案例二：画房子**

```scratch
定义 画房子 大小
// 画正方形（房子主体）
重复执行 4 次
    移动 大小 步
    右转 90 度
结束

// 画三角形（屋顶）
移动 大小 步
右转 30 度
重复执行 3 次
    移动 大小 步
    右转 120 度
结束
```

**案例三：递归画树**

```scratch
定义 画树枝 长度
如果 长度 > 5 那么
    移动 长度 步
    右转 20 度
    画树枝 长度 * 0.7
    左转 40 度
    画树枝 长度 * 0.7
    右转 20 度
    移动 负的长度 步
结束
```

## 📚 模块四：模块化编程

### 4.1 什么是模块化？

**把大程序拆成小模块**，每个模块负责一部分功能。

就像建房子：
- 打地基的人只管地基
- 砌墙的人只管砌墙
- 装修的人只管装修

**好处**：
- 分工明确
- 容易维护
- 可以复用

### 4.2 Scratch中的模块化

**方法**：用多个角色，每个角色负责一部分

**示例：完整游戏**

| 角色 | 职责 |
|:---|:---|
| 玩家 | 控制移动、发射子弹 |
| 敌人 | 移动、被击中判定 |
| 子弹 | 飞行、碰撞检测 |
| 分数显示 | 显示分数 |
| 背景控制 | 切换背景、游戏状态 |

### 4.3 广播通信

不同角色之间用广播通信：

```scratch
// 玩家角色
当按下空格键
广播 发射子弹

// 子弹角色
当接收到 发射子弹
克隆自己

// 敌人角色
当接收到 击中敌人
将得分增加 10
```

// 搜索联系人
当角色被点击

将找到设为 假

    如果 姓名的第计数器项 = 搜索名 那么
        说找到！电话是：电话的第计数器项 2秒

## 🎯 三经典智慧

### 《易经》· 物以类聚

**列表就是把同类的东西聚在一起。**

就像：
- 把苹果、香蕉放在一起（水果列表）
- 把所有学生的成绩放在一起（成绩列表）
- 把相同功能的代码放在一起（函数）

分类整理，是处理复杂问题的基础。

### 《道德经》· 道法自然

**函数就是顺应自然规律。**

自然界处处是函数：
- 输入阳光、水 → 输出果实（植物函数）
- 输入食物 → 输出能量（消化函数）

把重复的代码封装成函数，让程序像自然一样简洁高效。

### 《论语》· 工欲善其事，必先利其器

学编程要善用工具：
- 列表是存储数据的工具
- 函数是复用代码的工具
- 模块化是组织程序的工具

有了好工具，才能做出好作品。
