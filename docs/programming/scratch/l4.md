## 📚 知识模块

### 模块一：算法入门

#### 1.1 什么是算法？

**算法就是解决问题的方法和步骤。**

就像做一道菜的食谱：
1. 准备食材
2. 按顺序加工
3. 调味装盘

编程中的算法：
- 排序：把一组数从小到大排列
- 搜索：在一组数中找到目标
- 找最大/最小值

#### 1.2 算法的评价

**好的算法标准**：

| 标准 | 说明 |
|:---|:---|
| **正确性** | 能正确解决问题 |
| **效率** | 步骤少、速度快 |
| **可读性** | 容易理解和修改 |

**时间复杂度**：
- O(n)：线性时间，n 个数处理 n 步
- O(n²)：平方时间，n 个数处理 n×n 步
- O(log n)：对数时间，非常高效

### 模块二：排序算法

#### 2.1 冒泡排序

**思想**：相邻两个数比较，大的往后冒。

```
初始：[5, 3, 8, 1, 2]

第1轮：
  5和3比，交换 → [3, 5, 8, 1, 2]
  5和8比，不动 → [3, 5, 8, 1, 2]
  8和1比，交换 → [3, 5, 1, 8, 2]
  8和2比，交换 → [3, 5, 1, 2, 8]  （8到位）

第2轮：
  3和5比，不动 → [3, 5, 1, 2, 8]
  5和1比，交换 → [3, 1, 5, 2, 8]
  5和2比，交换 → [3, 1, 2, 5, 8]  （5到位）

...继续直到全部有序
```

**Scratch 实现思路**：
- 用列表存储数字
- 双重循环比较相邻项
- 用变量交换位置

#### 2.2 选择排序

**思想**：每次选最小的，放到前面。

```
初始：[5, 3, 8, 1, 2]

第1轮：找最小是1，和第1个交换 → [1, 3, 8, 5, 2]
第2轮：从第2开始找最小是2，交换 → [1, 2, 8, 5, 3]
第3轮：从第3开始找最小是3，交换 → [1, 2, 3, 5, 8]
...完成
```

**特点**：
- 比较次数固定
- 交换次数少
- 适合数据移动代价大的情况

### 模块三：搜索算法

#### 3.1 顺序搜索

**思想**：一个一个找，直到找到或找完。

```scratch
定义 顺序搜索 (目标)
将 位置 设为 1
重复执行直到 (位置) > (列表长度)
    如果 (列表的第(位置)项) = (目标) 那么
        返回 (位置)
    结束
    将 位置 增加 1
结束
返回 (0)  // 没找到
```

**时间复杂度**：O(n) —— 最坏要找 n 次

#### 3.2 二分搜索

**思想**：数据有序时，每次排除一半。

```
有序列表：[1, 3, 5, 7, 9, 11, 13, 15]
搜索：7

第1次：中间是9，7<9，在左半找 [1, 3, 5, 7]
第2次：中间是3，7>3，在右半找 [5, 7]
第3次：中间是5，7>5，在右半找 [7]
第4次：找到！

只需4次，比顺序搜索快多了！
```

**时间复杂度**：O(log n) —— 非常高效！

## 🎯 三经典智慧

### 《易经》· 穷则变，变则通

**排序的本质就是「变化」** —— 把无序变为有序。

不同的排序算法，就是不同的变化策略：
- 冒泡排序：相邻交换，逐步有序
- 选择排序：先找最小，确定位置

**穷则变**：当一种方法效率低时，就要换一种方法。

### 《道德经》· 以柔克刚

**二分搜索的智慧**：
- 不蛮力（顺序搜索）
- 用智慧（每次排除一半）
- 以少胜多（log n vs n）

### 《论语》· 工欲善其事，必先利其器

算法就是编程的「器」。
- 学会算法，事半功倍
- 选好算法，程序高效
