# 算法可视化：用 Scratch 演示算法

> 把抽象的算法变成看得见的动画

---

## 🎯 为什么需要算法可视化？

算法往往是抽象的，初学者难以理解。通过可视化：
- ✅ 看到算法的执行过程
- ✅ 理解数据的变化
- ✅ 发现算法的规律
- ✅ 更容易记住算法

---

## 📊 冒泡排序可视化

### 原理回顾

冒泡排序：相邻元素两两比较，大的往后"冒泡"

### Scratch 实现

```scratch
定义 冒泡排序可视化
    // 准备数据
    删除 数据 的全部项目
    重复执行 10 次
        将 随机数从 10 到 100 加入 数据
    结束重复
    
    // 绘制初始状态
    绘制柱状图
    等待 1 秒
    
    // 冒泡排序
    将 n 设为 数据 的长度
    对于 i 从 1 到 n-1
        对于 j 从 1 到 n-i
            // 高亮正在比较的两个数
            高亮柱子 j
            高亮柱子 j+1
            等待 0.5 秒
            
            // 比较
            如果 数据 的第 j 项 > 数据 的第 j+1 项 那么
                // 交换
                播放声音 交换
                交换 数据 的第 j 项 和 第 j+1 项
                重绘柱子 j 和 j+1
            结束如果
            
            // 取消高亮
            取消高亮
        结束对于
        
        // 标记已排序的部分
        标记已排序 n-i+1
    结束对于
    
    说 "排序完成！" 持续 2 秒
结束定义

定义 绘制柱状图
    擦除全部
    将 x 设为 -200
    对于 i 从 1 到 数据 的长度
        落笔
        将 y 坐标 设为 -150
        将 画笔颜色 设为 蓝色
        将 画笔粗细 设为 15
        移动 数据 的第 i 项 步
        抬笔
        将 x 坐标 增加 20
    结束对于
结束定义

定义 高亮柱子 接收 索引
    // 用不同颜色重绘该柱子
结束定义
```

### 学习要点

1. **观察冒泡过程**：
   - 最大的数如何一步步"冒"到最后
   - 每一轮确定一个数的位置

2. **理解时间复杂度**：
   - 为什么慢？需要看多少轮
   - 每轮比较多少次

## 🌳 二叉树遍历可视化

### 原理回顾

- **前序遍历**：根 → 左 → 右
- **中序遍历**：左 → 根 → 右
- **后序遍历**：左 → 右 → 根

### Scratch 实现

```scratch
// 二叉树节点数据结构
// 每个节点：值、左子节点索引、右子节点索引、x坐标、y坐标

定义 绘制树
    擦除全部
    // 递归绘制每个节点和连线
    绘制节点 1  // 从根节点开始
结束定义

定义 绘制节点 接收 节点索引
    如果 节点索引 = 0 那么 停止 这个脚本  // 空节点
    
    移到 x:节点x[节点索引] y:节点y[节点索引]
    盖章  // 绘制节点圆圈
    说 节点值[节点索引] 持续 0 秒  // 显示值
    
    // 绘制到左子节点的连线
    如果 左子节点[节点索引] > 0 那么
        落笔
        移到 x:节点x[左子节点[节点索引]] y:节点y[左子节点[节点索引]]
        抬笔
        绘制节点 左子节点[节点索引]
    结束如果
    
    // 绘制到右子节点的连线
    如果 右子节点[节点索引] > 0 那么
        落笔
        移到 x:节点x[节点索引] y:节点y[节点索引]
        移到 x:节点x[右子节点[节点索引]] y:节点y[右子节点[节点索引]]
        抬笔
        绘制节点 右子节点[节点索引]
    结束如果
结束定义

// 前序遍历可视化
定义 前序遍历 接收 节点索引
    如果 节点索引 = 0 那么 停止 这个脚本
    
    // 访问根节点（高亮）
    高亮节点 节点索引
    说 连接 "访问：" 节点值[节点索引] 持续 1 秒
    将 节点值[节点索引] 加入 遍历结果
    
    // 遍历左子树
    前序遍历 左子节点[节点索引]
    
    // 遍历右子树
    前序遍历 右子节点[节点索引]
结束定义
```

## 🔍 二分查找可视化

### 原理回顾

在有序数组中，每次排除一半，快速找到目标

### Scratch 实现

```scratch
定义 二分查找可视化 接收 目标值
    将 左边界 设为 1
    将 右边界 设为 数据 的长度
    
    重复执行直到 左边界 > 右边界
        // 计算中间位置
        将 中间 设为 (左边界 + 右边界) / 2
        
        // 可视化：高亮当前查找范围
        高亮范围 左边界 右边界
        高亮中间 中间
        等待 1 秒
        
        // 比较
        如果 数据 的第 中间 项 = 目标值 那么
            说 连接 "找到了！位置：" 中间 持续 2 秒
            停止 这个脚本
        结束如果
        
        如果 数据 的第 中间 项 > 目标值 那么
            说 "中间值太大，查左半边" 持续 1 秒
            将 右边界 设为 中间 - 1
        否则
            说 "中间值太小，查右半边" 持续 1 秒
            将 左边界 设为 中间 + 1
        结束如果
    结束重复
    
    说 "未找到！" 持续 2 秒
结束定义
```

## 🧭 迷宫生成与求解可视化

### 迷宫生成（递归回溯）

```scratch
定义 生成迷宫 接收 x y
    将 迷宫[x][y] 设为 通路
    标记已访问 x y
    
    // 随机打乱四个方向
    随机化方向列表
    
    对于 每个方向
        将 新x 设为 x + 方向x * 2
        将 新y 设为 y + 方向y * 2
        
        如果 新x 在范围内 且 新y 在范围内 且 未访问 新x 新y 那么
            // 打通墙壁
            将 墙壁[x+方向x][y+方向y] 设为 通路
            绘制通路动画
            
            // 递归
            生成迷宫 新x 新y
        结束如果
    结束对于
结束定义
```

### 迷宫求解（BFS）

```scratch
定义 BFS求解迷宫
    清空队列
    将起点加入队列
    标记起点距离为0
    
    重复执行直到 队列为空
        将 当前 设为 出队
        
        // 可视化：高亮当前探索位置
        高亮格子 当前
        等待 0.1 秒
        
        如果 当前 = 终点 那么
            显示路径
            停止 这个脚本
        结束如果
        
        // 探索四个方向
        对于 每个方向
            将 邻居 设为 当前 + 方向
            如果 可通行 邻居 且 未访问 邻居 那么
                标记 邻居 距离 = 当前距离 + 1
                记录 邻居 的前驱 = 当前
                入队 邻居
                
                // 可视化：标记待探索
                标记待探索 邻居
            结束如果
        结束对于
    结束重复
结束定义
```

## 🎨 可视化技巧

### 1. 颜色编码

```
未排序：蓝色
正在比较：黄色
需要交换：红色
已排序：绿色
```

### 2. 速度控制

```scratch
定义 设置速度 接收 速度级别
    如果 速度级别 = "慢" 那么
        将 等待时间 设为 1
    否则 如果 速度级别 = "中" 那么
        将 等待时间 设为 0.5
    否则
        将 等待时间 设为 0.1
    结束如果
结束定义
```

### 3. 步骤说明

```scratch
定义 显示步骤说明 接收 说明
    说 说明 持续 2 秒
    // 同时在屏幕上显示文字
    创建文字 说明 在位置 (x, y)
结束定义
```

## 🎯 教学建议

### 使用可视化教学的步骤

1. **先演示**：运行可视化，学生观察
2. **再讲解**：解释每一轮在做什么
3. **让学生描述**：学生用自己的话描述过程
4. **手动模拟**：用纸笔模拟算法
5. **写代码**：最后自己实现

### 常见问题

**Q：可视化会不会让学生只看不思考？**

A：关键是引导：
- 暂停提问："猜猜下一步会发生什么？"
- 让学生预测结果
- 比较不同算法的可视化
