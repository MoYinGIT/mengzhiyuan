## 【大题1】复合运算符的基础原理

**考察点**：理解复合运算符的本质是"先运算，再赋值"

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 10, b = 10;
    
    a = a + 5;      // 语句1：普通赋值
    b += 5;         // 语句2：复合赋值
    
    cout << "a=" << a << ", b=" << b << endl;
    
    int x = 7;
    x -= 3;         // 语句3
    
    int y = 8;
    y *= 2;         // 语句4
    
    cout << "x=" << x << ", y=" << y << endl;
    return 0;
}
```

### 一、选择题

**1. 语句1和语句2的关系是（ ）**

A. 完全不同，结果不同  
B. 效果完全相同，但复合赋值更高效  
C. 语句2是语句1的简写，效果相同  
D. 语句2有副作用，结果不同

**2. 语句3执行后，x的值是（ ）**

A. 10  
B. 4  
C. 3  
D. 7

**3. `b += 5` 等价于（ ）**

A. `b = b + 5`  
B. `b = 5 + b`  
C. A和B都对  
D. `b =+ 5`

### 二、判断题

**4. 复合运算符`+=`的作用是将右边表达式的值加到左边变量上，然后将结果赋给左边变量。**  
（ ）正确 （ ）错误

**5. `a += b` 和 `a = a + b` 在效果上是完全等价的，可以任意替换。**  
（ ）正确 （ ）错误

**6. 复合运算符比普通赋值写法更简洁，且可能更高效（因为左边变量只计算一次）。**  
（ ）正确 （ ）错误

### 三、填空题

**7. 复合运算符`+=`、`-=`、`*=`、`/=`、`%=`的作用是先将左边变量与右边表达式进行______运算，然后将结果______给左边变量。**

**8. `a += 5` 等价于 `a = ______ + 5`。**

**9. 如果a=10，执行`a += 5`后，a的值是 ______。**

**7. 复合赋值运算符的优先级比算术运算符 ______（高/低）。**

#include <iostream>

}

B. 4

## 复合运算符速查表

| 运算符 | 等价于 | 说明 | 示例 |
|:---:|:---|:---|:---:|
| `+=` | `a = a + b` | 加后赋值 | `a += 5` → `a = a + 5` |
| `-=` | `a = a - b` | 减后赋值 | `a -= 3` → `a = a - 3` |
| `*=` | `a = a * b` | 乘后赋值 | `a *= 2` → `a = a * 2` |
| `/=` | `a = a / b` | 除后赋值 | `a /= 4` → `a = a / 4` |
| `%=` | `a = a % b` | 取模后赋值 | `a %= 3` → `a = a % 3` |
| `<<=` | `a = a << b` | 左移后赋值 | `a <<= 1` → `a = a * 2` |
| `>>=` | `a = a >> b` | 右移后赋值 | `a >>= 1` → `a = a / 2` |
| `&=` | `a = a & b` | 按位与后赋值 | `a &= 1` → 保留最低位 |
| `|=` | `a = a | b` | 按位或后赋值 | `a |= 1` → 设置最低位 |
| `^=` | `a = a ^ b` | 按位异或后赋值 | `a ^= 1` → 翻转最低位 |

## 核心要点总结

### 复合运算符使用检查清单

- [ ] `a += b` 等价于 `a = a + b`，更简洁
- [ ] 复合运算符优先级低于算术运算符
- [ ] `a *= b + c` 等价于 `a = a * (b + c)`
- [ ] 整数除法截断小数，`/=` 和 `%=` 配合使用
- [ ] 位运算复合赋值：`<<=`乘2，`>>=`除2
- [ ] 复合赋值时会发生类型转换，保持左边变量类型
- [ ] 连续复合赋值是右结合的
- [ ] 常用于累加、累乘、计数等场景
- [ ] 可以用于数组元素和指针指向的值

### 常见错误对比

| 错误写法 | 问题 | 正确理解 |
|:---|:---|:---|
| `a *= 2 + 3` 期望得 `a*2+3` | 优先级问题 | 实际是 `a = a * (2+3)` |
| `int a = 5; a += 2.5` 期望得7.5 | 类型截断 | 实际是7（int类型） |
| `a += b *= c` 不清楚顺序 | 右结合 | 先算`b*=c`，再算`a+=b` |
| 累乘初始化为0 | 结果永远是0 | 应该初始化为1 |
