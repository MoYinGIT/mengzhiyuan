## 📚 知识探索

### 🔍 探索1：编程是什么？

**想象一下**：你有一只聪明的小狗，你想让它坐下、握手、转圈，你会怎么做？

你会：
1. 说出指令（"坐下"）
2. 小狗听懂并执行
3. 小狗坐下

**编程就像这样**！只不过我们不是跟小狗说话，而是跟计算机说话。

**生活类比**：
- 编程 = 写菜谱（告诉厨师每一步怎么做）
- 程序 = 一道菜（按照菜谱做出来的成品）
- 计算机 = 厨师（按照指令做事）

---

### 🔍 探索2：认识Scratch的"脸蛋"

**Scratch就像一个大积木盒**，里面有五颜六色的积木块，每种颜色代表不同的功能。

**找到这五个区域**：

| 区域 | 位置 | 作用 |
|:---|:---|:---|
| **积木区** | 左边 | 存放各种积木块（就像积木盒） |
| **代码区** | 中间 | 拼积木的地方（就像搭积木的桌子） |
| **舞台区** | 右上角 | 程序运行的地方（就像表演的舞台） |
| **角色区** | 舞台下方 | 显示所有角色 |
| **菜单栏** | 最上面 | 保存、打开文件 |

**试一试**：
1. 打开 [scratch.mit.edu](https://scratch.mit.edu)
2. 点击"创建"进入Scratch编辑器
3. 数一数，积木区有几种颜色的分类？

<details>
<summary>💡 需要提示？</summary>

**提示**：看看积木区最左边，有「运动」「外观」「声音」「事件」「控制」等标签，每个标签就是一种颜色！
</details>

### 🔍 探索3：积木怎么玩？

**Scratch的积木就像乐高**，有凸起的接口和凹陷的接口，只有形状匹配才能拼在一起！

**积木的形状有秘密**：
- 🔷 **帽子形**（顶部圆弧）：程序的起点，只能放在最上面
- 🧩 **拼图形**（上下都有凸起/凹陷）：中间的步骤，可以连接其他积木
- ⭕ **椭圆形**：代表数据，比如数字、文字
- 🔶 **六边形**：代表判断，比如"是否碰到了边缘"

**试一试**：
1. 从「事件」分类中，拖出「当绿旗被点击」到代码区
2. 从「外观」分类中，拖出「说 [你好] 2秒」
3. 把两个积木拼在一起
4. 点击舞台区上方的🟢绿旗

**发生了什么？** 🤔

<details>
<summary>💡 需要提示？</summary>

**提示**：如果拼不上，看看积木的接口形状是否匹配。就像乐高积木，凸的要对凹的！
</details>

### 🔍 探索4：坐标——舞台的"座位号"

**生活场景**：去电影院看电影，你的票上写着"5排8号"，这就是你的座位位置。

**Scratch的舞台也有"座位号"**，叫做**坐标**：
- **x坐标**：左右位置（像排号）
  - 0 = 正中间
  - 正数 = 右边（1, 2, 3... 越来越靠右）
  - 负数 = 左边（-1, -2, -3... 越来越靠左）
  
- **y坐标**：上下位置（像座位号）
  - 0 = 正中间
  - 正数 = 上面
  - 负数 = 下面

**舞台的边界**：
- 最左边：x = -240
- 最右边：x = 240
- 最上面：y = 180
- 最下面：y = -180

**想一想**：舞台中心点的坐标是多少？

<details>
<summary>✅ 答案</summary>

中心点是 **x: 0, y: 0**
</details>

## ✍️ 动手实践

### 🟢 练习1：小猫打招呼（基础）

**要求**：让小猫说"你好，我是小猫！"，然后移动到舞台中心。

**Step 1**：添加「当绿旗被点击」积木
**Step 2**：添加「说 [你好，我是小猫！] 2秒」
**Step 3**：添加「移到 x: (0) y: (0)」

**完整代码**：
```scratch
当绿旗被点击
说 [你好，我是小猫！] 2秒
移到 x: (0) y: (0)
```

<details>
<summary>✅ 参考答案截图</summary>

积木顺序：
1. 事件 - 当绿旗被点击
2. 外观 - 说 [...] 2秒
3. 运动 - 移到 x: (...) y: (...)

</details>

**提示**：需要三个「说...2秒」积木

<details>
<summary>✅ 参考答案</summary>

```scratch
当绿旗被点击
说 [你好！] 2秒
说 [我今年3岁了] 2秒
说 [我喜欢吃鱼！] 2秒
```

</details>

### 🟡 练习3：小猫去散步（进阶）

**要求**：让小猫从左边走到右边。

**分解步骤**：
- Step 1：先把小猫移到左边（x: -200, y: 0）
- Step 2：然后让它一步一步往右走（用「移动 (10) 步」）
- Step 3：重复走多次

**想一想**：要让小猫从-200走到200，每次走10步，需要走几次？

<details>
<summary>✅ 参考答案</summary>

```scratch
当绿旗被点击
移到 x: (-200) y: (0)
重复执行 (40) 次
    移动 (10) 步
    等待 (0.1) 秒
结束
```

**解释**：从-200到200，距离是400。每次走10步，需要走 400÷10 = 40 次。

</details>

---

### 🟡 练习4：小猫瞬移（进阶）

**要求**：让小猫先出现在左上角，等待1秒后，瞬间移动到右下角。

**提示**：
- 左上角坐标：x = -200, y = 150
- 右下角坐标：x = 200, y = -150
- 使用「等待 (1) 秒」积木

<details>
<summary>✅ 参考答案</summary>

```scratch
当绿旗被点击
移到 x: (-200) y: (150)
说 [我在左上角！] 1秒
等待 (1) 秒
移到 x: (200) y: (-150)
说 [我到了右下角！] 2秒
```

</details>

</details>

## 🧩 综合应用

### 题目：小猫的舞台表演

**场景**：小猫要表演一个节目，流程如下：
1. 从舞台左边慢慢走到中心（用2秒）
2. 停在中心说"大家好！"2秒
3. 继续说"我要表演转圈！"2秒
4. 向右旋转360度（转一圈）
5. 说"表演结束，谢谢大家！"2秒
6. 退到舞台右边

**想一想**：
- 用什么积木可以让小猫慢慢移动？
- 用什么积木可以让小猫旋转？

<details>
<summary>✅ 参考答案</summary>

```scratch
当绿旗被点击
移到 x: (-200) y: (0)
在 (2) 秒内滑行到 x: (0) y: (0)
说 [大家好！] 2秒
说 [我要表演转圈！] 2秒
重复执行 (36) 次
    右转 (10) 度
    等待 (0.1) 秒
结束
说 [表演结束，谢谢大家！] 2秒
在 (2) 秒内滑行到 x: (200) y: (0)
```

**积木说明**：
- 「滑行到」可以让角色平滑移动
- 36次×10度 = 360度，正好转一圈

</details>

### 🐛 常见错误自查

| 错误 | 原因 | 解决 |
|:---|:---|:---|
| 积木拼不上 | 接口形状不匹配 | 检查积木的凸起和凹陷 |
| 点击绿旗没反应 | 缺少「当绿旗被点击」 | 加上这个积木作为起点 |
| 角色跑不见了 | 坐标设置错误 | 用「移到 x:0 y:0」找回 |
| 说话太快看不清 | 等待时间太短 | 增加「等待」积木或延长说话时间 |
