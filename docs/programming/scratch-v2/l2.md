## 📚 知识探索

### 🔍 探索1：事件——程序的"开关"

**生活类比**：
- 闹钟响了 → 你起床（闹钟是"事件"，起床是"响应"）
- 门铃响了 → 去开门（门铃是"事件"，开门是"响应"）
- 妈妈喊吃饭了 → 去吃饭（喊话是"事件"，吃饭是"响应"）

**编程中的事件**：告诉程序"什么时候开始做"。

**Scratch里的事件积木**（黄色，帽子形状）：

| 积木 | 触发条件 | 生活例子 |
|:---|:---|:---|
| 当绿旗被点击 🟢 | 点击绿旗 | 老师说"开始上课" |
| 当角色被点击 🖱️ | 点击角色 | 有人拍你肩膀 |
| 当按下 [空格] 键 | 按键盘 | 按门铃 |
| 当接收到 [消息] | 收到广播 | 听到广播通知 |

**试一试**：
1. 拖出「当角色被点击」积木
2. 下面连接「说 [你点到我啦！] 2秒」
3. 点击舞台上的小猫，看看发生什么

### 🔍 探索2：方向与角度

**生活场景**：你在操场上，老师让你"面向东"、"向右转"，你怎么做？

**Scratch的角度**：
- 0° = 向上（朝向天空）
- 90° = 向右（默认方向）
- 180° = 向下
- -90°（或270°）= 向左

**记忆口诀**：
> 上0右90，下180左270

**控制方向的积木**：

| 积木 | 作用 |
|:---|:---|
| 面向 (90) 方向 | 直接设定面向的角度 |
| 右转 (15) 度 | 向右转一定角度 |
| 左转 (15) 度 | 向左转一定角度 |
| 面向 [鼠标指针] | 始终面向鼠标 |

**想一想**：让小猫面向左边，应该设置多少度？

<details>
<summary>💡 需要提示？</summary>

**提示**：左边是-90度或270度，两种写法都可以！
</details>

### 🔍 探索3：用键盘控制角色

**现在我们来做一个游戏**：用方向键控制小猫移动！

**核心原理**：
- 按下"向上键" → 小猫向上移动
- 按下"向左键" → 小猫向左移动

**试一试**：
1. 从「事件」中找到「当按下 [空格] 键」
2. 把"空格"改成"向上箭头"
3. 下面连接：
   - 「面向 (0) 方向」（向上）
   - 「移动 (10) 步」
4. 按下键盘上的↑键，看看小猫动不动

**发现规律了吗？**
- 每个方向键都需要单独的一组积木
- 先设置面向的方向，再移动

### 🔍 探索4：外观积木——让角色更生动

**生活类比**：你在表演话剧，不仅要说话，还要有表情、动作、换装！

**外观积木分类**：

#### 说话与思考
| 积木 | 效果 |
|:---|:---|
| 说 [你好] 2秒 | 气泡对话框 |
| 思考 [嗯...] 2秒 | 思考气泡 |

#### 大小与显示
| 积木 | 效果 |
|:---|:---|
| 将大小设为 (100) | 改变大小（100是原大小）|
| 将大小增加 (10) | 变大一点 |
| 显示 / 隐藏 | 显示或隐藏角色 |

#### 颜色与特效
| 积木 | 效果 |
|:---|:---|
| 将 [颜色] 特效增加 (25) | 变色 |
| 将 [亮度] 特效增加 (25) | 变亮/变暗 |
| 将 [虚像] 特效增加 (25) | 变透明（虚像=100时消失）|
| 清除图形特效 | 恢复原状 |

#### 造型切换
| 积木 | 效果 |
|:---|:---|
| 下一个造型 | 切换到下一个姿势 |
| 换成 [造型1] 造型 | 切换到指定造型 |

**试一试**：
1. 拖出「将颜色特效增加 (25)」
2. 多点击几次，看看小猫变成什么颜色
3. 点击「清除图形特效」恢复原状

---

### 🔍 探索5：制作走路动画

**原理**：快速切换不同造型，就像翻书动画！

**小猫有两个造型**：
- 造型1：四条腿并拢
- 造型2：腿张开

**快速切换 → 看起来像走路**

**试一试**：
```scratch
当绿旗被点击
重复执行
    下一个造型
    移动 (10) 步
    等待 (0.2) 秒
    如果 <碰到边缘？> 那么
        碰到边缘就反弹
    结束
结束
```

**发现规律了吗？**
- 「下一个造型」让小猫切换姿势
- 「等待」让动作不要太快，否则看不清
- 「碰到边缘就反弹」防止小猫跑出舞台

## ✍️ 动手实践

### 🟢 练习1：四方向移动（基础）

**要求**：用↑↓←→四个方向键控制小猫移动。

**提示**：需要四组积木，每组对应一个方向键。

<details>
<summary>✅ 参考答案</summary>

```scratch
// 向上
当按下 [向上 v] 键
面向 (0) 方向
移动 (10) 步

// 向下
当按下 [向下 v] 键
面向 (180) 方向
移动 (10) 步

// 向左
当按下 [向左 v] 键
面向 (-90) 方向
移动 (10) 步

// 向右
当按下 [向右 v] 键
面向 (90) 方向
移动 (10) 步
```

</details>

### 🟢 练习2：点击互动（基础）

**要求**：点击小猫时，它会说"你点到我啦！"并变大一点。

**分解步骤**：
- Step 1：使用「当角色被点击」事件
- Step 2：说一句话
- Step 3：使用「将大小增加 (20)」

<details>
<summary>✅ 参考答案</summary>

```scratch
当角色被点击
说 [你点到我啦！] 2秒
将大小增加 (20)
```

**注意**：多点击几次，小猫会变得非常大！

</details>

### 🟡 练习3：变色龙小猫（进阶）

**要求**：每按一次空格键，小猫就随机变色。

**提示**：
- 使用「当按下 [空格] 键」事件
- 颜色特效的随机范围是0到200

<details>
<summary>💡 需要提示？</summary>

**提示**：
- 「将颜色特效设为 (随机数 (0) 到 (200))」
- 或者使用「将颜色特效增加 (25)」每次变一点
</details>

<details>
<summary>✅ 参考答案</summary>

```scratch
当按下 [空格 v] 键
将 [颜色 v] 特效设为 (随机数 (0) 到 (200))
```

</details>

### 🟡 练习4：呼吸效果（进阶）

**要求**：让小猫像呼吸一样，慢慢变大再慢慢变小，循环往复。

**分解步骤**：
- Step 1：先变大（重复增加大小）
- Step 2**：再变小（重复减小大小）
- Step 3：放进「重复执行」里循环

<details>
<summary>✅ 参考答案</summary>

```scratch
当绿旗被点击
将大小设为 (100)
重复执行
    重复执行 (20) 次
        将大小增加 (5)
        等待 (0.05) 秒
    结束
    重复执行 (20) 次
        将大小增加 (-5)
        等待 (0.05) 秒
    结束
结束
```

**解释**：
- 外层「重复执行」让呼吸永远进行
- 内层第一个循环让猫变大（从100到200）
- 内层第二个循环让猫变小（从200回到100）
- 增加负数 = 减小

</details>

### 🔴 练习5：魔法小猫（挑战）

**要求**：制作一个魔法小猫：
1. 按空格键：随机变色
2. 按上键：变大
3. 按下键：变小
4. 点击小猫：消失（虚像=100）并说"拜拜~"
5. 按R键：重新出现（虚像=0）

**想一想**：需要几组事件积木？

<details>
<summary>💡 需要提示？</summary>

**提示**：
1. 变色：「将颜色特效设为 (随机数)」
2. 变大：「将大小增加 (10)」
3. 变小：「将大小增加 (-10)」
4. 消失：「将虚像特效设为 (100)」+ 说拜拜
5. 重新出现：「将虚像特效设为 (0)」

需要5组事件积木！
</details>

<details>
<summary>✅ 参考答案</summary>

```scratch
// 1. 空格键变色
当按下 [空格 v] 键
将 [颜色 v] 特效设为 (随机数 (0) 到 (200))

// 2. 上键变大
当按下 [向上 v] 键
将大小增加 (10)

// 3. 下键变小
当按下 [向下 v] 键
将大小增加 (-10)

// 4. 点击消失
当角色被点击
将 [虚像 v] 特效设为 (100)
说 [拜拜~] 2秒

// 5. R键重新出现
当按下 [r v] 键
将 [虚像 v] 特效设为 (0)
说 [我又回来啦！] 2秒
```

</details>

## 🧩 综合应用

### 题目：小猫捉迷藏

**场景**：
1. 点击绿旗，小猫出现在随机位置，说"来抓我呀！"
2. 玩家用方向键控制小猫移动
3. 点击小猫时，小猫说"被你抓到啦！"然后随机瞬移到新位置
4. 如果小猫碰到舞台边缘，说"哎哟！"并反弹

**想一想**：
- 如何实现"随机位置"？
- 需要检测哪些事件？

<details>
<summary>💡 需要提示？</summary>

**提示**：
- 随机位置：「移到 x: (随机数 (-200) 到 (200)) y: (随机数 (-150) 到 (150))」
- 需要的事件：当绿旗被点击、当角色被点击、当按下方向键
- 边缘检测：使用「如果 <碰到边缘？>」
</details>

<details>
<summary>✅ 参考答案</summary>

```scratch
// 初始化
当绿旗被点击
移到 x: (随机数 (-200) 到 (200)) y: (随机数 (-150) 到 (150))
说 [来抓我呀！] 2秒

// 被点击时瞬移
当角色被点击
说 [被你抓到啦！] 1秒
移到 x: (随机数 (-200) 到 (200)) y: (随机数 (-150) 到 (150))

// 方向键控制（略，参考练习1）

// 边缘检测（可以加到移动代码中）
如果 <碰到边缘？> 那么
    说 [哎哟！] 0.5秒
    碰到边缘就反弹
结束
```

</details>

### 🐛 常见错误自查

| 错误 | 原因 | 解决 |
|:---|:---|:---|
| 按方向键没反应 | 选错了事件积木 | 确认使用「当按下 [ ] 键」 |
| 角色只转圈不移动 | 缺少「移动」积木 | 加上「移动 (10) 步」 |
| 角色移动方向不对 | 面向角度错了 | 检查「面向」的角度值 |
| 颜色特效不恢复 | 没有清除特效 | 用「清除图形特效」恢复 |
| 走路动画太快 | 缺少等待 | 加上「等待 (0.1) 秒」 |
