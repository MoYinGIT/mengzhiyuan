## 贡献方式

### 方式一：提交题解

#### 什么是好题解？

| 要素 | 说明 | 示例 |
|:---|:---|:---|
| **清晰的思路** | 先讲思路，再给代码 | "这道题可以用贪心算法..." |
| **详细的注释** | 关键步骤要有注释 | `// 排序后贪心选择` |
| **复杂度分析** | 时间和空间复杂度 | 时间O(nlogn)，空间O(n) |
| **易错点提示** | 常见错误和避免方法 | "注意边界情况..." |
| **多种解法** | 如果有多种方法，都介绍 | 暴力→优化→最优 |

#### 题解格式模板

```markdown
# P1001 A+B Problem

## 题目简述
一句话概括题目要求。

## 解题思路
1. 读入两个整数
2. 计算它们的和
3. 输出结果

## 参考代码

```cpp
#include <iostream>
using namespace std;

int main() {
    int a, b;
    cin >> a >> b;  // 读入两个数
    cout << a + b << endl;  // 输出和
    return 0;
}
```

## 复杂度分析
- 时间复杂度：O(1)
- 空间复杂度：O(1)

## 易错点
- 注意输出格式，不要有多余空格
- 注意数据范围，使用合适的数据类型

## 拓展思考
如果数据范围很大（a,b < 10^100），该怎么做？
```

#### 提交方式

1. **Fork** 本仓库
2. 在 `docs/solutions/` 目录下创建你的题解文件
3. 按照模板格式编写
4. 提交 **Pull Request**
5. 等待审核合并

### 方式二：改进文档

#### 可以改进什么？

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| **纠错** | 发现错别字、错误内容 | 修正代码错误 |
| **补充** | 内容不够详细 | 添加更多例题 |
| **优化** | 表达可以更清晰 | 重写复杂段落 |
| **翻译** | 翻译优质英文资源 | 翻译英文题解 |
| **新增** | 创建新的教学内容 | 新增专题教程 |

#### 文档改进流程

```
1. 发现问题 / 想到改进点
        ↓
2. Fork 仓库到个人账户
        ↓
3. 创建分支（建议命名：fix-xxx 或 add-xxx）
        ↓
4. 修改文档
        ↓
5. 本地预览（npm run docs:dev）
        ↓
6. 提交 Commit（写清楚改动内容）
        ↓
7. 发起 Pull Request
        ↓
8. 等待 Review 和 Merge
```

## 贡献规范

### 文件命名规范

| 类型 | 命名格式 | 示例 |
|:---|:---|:---|
| 题解 | `题目ID-题目名.md` | `p1001-a-plus-b.md` |
| 教程 | `主题名-guide.md` | `recursion-guide.md` |
| 文档 | `描述性名称.md` | `faq.md` |

### 代码规范

- 使用 **C++17** 标准
- 缩进使用 **4个空格**（或Tab）
- 变量名优先使用 **小驼峰命名法**（如 `studentName`, `totalScore`）
- 下划线命名法可接受但不优先（如 `student_name`）
- 关键步骤添加注释
- 不要出现中文符号（分号、括号等）

### 文档规范

- 使用 **Markdown** 格式
- 标题使用 `#` `##` `###` 层级
- 表格使用标准Markdown表格（不要用ASCII艺术）
- 代码块标明语言（```cpp）
- 图片放在 `docs/public/images/` 目录

### 禁止事项

❌ **禁止内容**：
- 抄袭内容（必须原创或注明出处）
- 恶意代码
- 广告推广
- 违反法律法规的内容
- 泄露他人隐私

## 快速开始

### 步骤1：Fork 仓库

1. 访问 [github.com/MoYinGIT/mengzhiyuan](https://github.com/MoYinGIT/mengzhiyuan)
2. 点击右上角 **"Fork"** 按钮
3. 等待Fork完成

### 步骤2：克隆到本地

```bash
git clone https://github.com/你的用户名/mengzhiyuan.git
cd mengzhiyuan
```

### 步骤3：安装依赖

```bash
cd docs
npm install
```

### 步骤4：本地预览

```bash
npm run docs:dev
```

访问 `http://localhost:5173` 查看效果

### 步骤5：创建分支

```bash
git checkout -b add-my-solution
```

### 步骤6：修改并提交

```bash
# 修改文件...
git add .
git commit -m "add: 添加P1001题解"
git push origin add-my-solution
```

### 步骤7：发起 Pull Request

1. 访问你的Fork仓库
2. 点击 **"Compare & pull request"**
3. 填写PR标题和描述
4. 点击 **"Create pull request"**

## 常见问题 FAQ

**Q1: 我不会Git，可以贡献吗？**
- 可以！可以先在 Issues 或 Discussions 发帖
- 也可以学习Git基础，推荐：[Git教程](https://www.liaoxuefeng.com/wiki/896043488029600)

**Q2: 我的水平不够，能贡献吗？**
- 完全可以！分享你的学习过程也很有价值
- 即使是简单的题解，对初学者也很有帮助
- "教学相长"，分享的过程也是学习的过程

**Q3: 贡献会被拒绝吗？**
- 只要内容有价值，格式符合规范，通常会被接受
- 如果有问题，我们会给出修改建议
- 不要因为担心被拒绝而不敢尝试

**Q4: 可以有署名吗？**
- 当然可以！题解末尾可以加上作者信息
- 格式：`**作者**：你的名字`

**Q5: 发现有错误但不知道怎么改？**
- 提交Issue说明问题
- 或在 Discussions 讨论
- 社区会一起解决
