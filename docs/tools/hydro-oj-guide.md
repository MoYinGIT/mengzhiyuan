# 🎯 Hydro OJ 在线评测使用指南

> 学生版 + 教师版完整教程

---

## 什么是 Hydro OJ？

**Hydro OJ** 是一个开源的在线评测系统（Online Judge），用于：
- 📝 在线编程练习
- 🏆 举办编程竞赛
- 📊 自动评测代码
- 📈 跟踪学习进度

**官方网站**: https://hydro.ac/

---

## 第一部分：学生版指南

### 1.1 注册与登录

#### 步骤1：访问平台
打开浏览器，访问 https://hydro.ac/ 或学校提供的 Hydro OJ 地址

#### 步骤2：注册账号
1. 点击右上角 **"注册"**
2. 填写信息：
   - 用户名（建议使用英文名，如 zhangsan）
   - 邮箱
   - 密码
3. 点击 **"注册"** 完成

#### 步骤3：完善资料
1. 登录后点击头像 → **"设置"**
2. 修改：
   - 昵称（显示名称）
   - 学校信息
   - 年级

### 1.3 做题流程

#### 步骤1：选择题目
1. 点击顶部 **"题库"**
2. 浏览题目列表，或使用筛选：
   - 难度：入门/普及/提高/省选/NOI
   - 标签：动态规划、图论等
   - 来源：GESP、NOIP等

#### 步骤2：阅读题目
点击题目名称，查看：
- **题目描述**: 问题背景和要求
- **输入格式**: 如何读取数据
- **输出格式**: 如何输出结果
- **样例输入/输出**: 示例数据
- **数据范围**: 时间和空间限制

**阅读技巧**:
- 先看输入输出格式
- 再看样例理解题意
- 最后看数据范围确定算法

#### 步骤3：编写代码
1. 在右侧编辑器选择语言（C++14/17/20）
2. 编写代码
3. 可以使用 **"格式化"** 美化代码

**代码模板**:
```cpp
#include <iostream>
using namespace std;

int main() {
    // 你的代码
    return 0;
}
```

#### 步骤4：提交评测
1. 点击 **"提交"** 按钮
2. 等待评测结果

---

### 1.4 评测结果解读

| 结果 | 含义 | 解决方案 |
|:---:|:---|:---|
| 🟢 **Accepted (AC)** | 通过！答案正确 | 恭喜，继续下一题 |
| 🔴 **Wrong Answer (WA)** | 答案错误 | 检查算法逻辑，重新读题 |
| ⏱️ **Time Limit Exceeded (TLE)** | 超时 | 优化算法，降低复杂度 |
| 💾 **Memory Limit Exceeded (MLE)** | 超内存 | 优化空间使用 |
| 💥 **Runtime Error (RE)** | 运行时错误 | 检查数组越界、除以零等 |
| ❌ **Compile Error (CE)** | 编译错误 | 检查语法错误 |
| ⚠️ **Presentation Error (PE)** | 格式错误 | 检查空格、换行 |
| 🚫 **Output Limit Exceeded (OLE)** | 输出超限 | 检查死循环输出 |

### 1.5 调试技巧

#### 技巧1：使用样例测试
在 **"自测"** 标签页：
1. 粘贴样例输入
2. 运行查看输出
3. 对比期望结果

#### 技巧2：添加调试输出
```cpp
// 使用 cerr 输出调试信息
cerr << "Debug: x = " << x << endl;

// 提交前删除或注释掉
```

#### 技巧3：边界数据测试
- 最小值：n=1
- 最大值：n=题目上限
- 特殊值：全0、全负数

#### 技巧4：查看错误信息
点击评测记录的 **"详情"** 查看：
- 错误点位置
- 实际输出 vs 期望输出
- 运行时间和内存

---

### 1.6 查看题解

1. 在题目页面点击 **"题解"** 标签
2. 查看他人解法
3. **注意**: 先自己思考，再看题解

**题解阅读方法**:
- 先理解思路
- 再看代码实现
- 最后自己写一遍

---

### 1.7 参与比赛

#### 步骤1：查看比赛
1. 点击顶部 **"比赛"**
2. 查看：
   - 正在进行
   - 即将开始
   - 已结束

#### 步骤2：报名参赛
1. 点击比赛名称
2. 点击 **"报名"**
3. 等待比赛开始

#### 步骤3：比赛期间
- 比赛开始后刷新页面
- 在题目列表中选择题目
- 提交代码
- 实时查看排名

**比赛规则**:
- 通常有罚时（WA一次+20分钟）
- 按通过题数排名，题数相同按时间
- 可以多次提交，以最后通过为准

### 1.8 学生常见问题 FAQ

**Q1: 为什么我的代码本地能跑，提交却WA？**
- 可能是边界情况没考虑到
- 输入输出格式有问题
- 使用了本地特有的特性

**Q2: 如何查看具体的测试数据？**
- 通常测试数据是隐藏的
- 可以下载错误点查看
- 或者自己构造类似数据

**Q3: 提交后一直"等待评测"怎么办？**
- 评测队列繁忙，请耐心等待
- 刷新页面查看状态
- 通常不超过1分钟

**Q4: 可以复制别人的代码吗？**
- ❌ 绝对禁止！会被判作弊
- 可以学习思路，但必须自己写
- 系统有代码查重

**Q5: 如何提高做题速度？**
- 熟练掌握STL
- 背熟常用算法模板
- 多练习，提高代码熟练度

## 第二部分：教师版指南

### 2.1 创建课程/训练

#### 步骤1：进入管理后台
1. 登录管理员账号
2. 点击头像 → **"管理后台"**

#### 步骤2：创建训练计划
1. 点击 **"训练"** → **"创建训练"**
2. 填写信息：
   - 训练名称：如 "GESP L3 循环专题"
   - 描述：训练目标和说明
   - 题目：从题库中添加
3. 点击 **"保存"**

#### 步骤3：设置权限
- 公开：所有人可见
- 私有：只有指定用户可见
- 密码保护：需要密码加入

---

### 2.2 布置作业

#### 步骤1：创建作业
1. 进入 **"作业"** 管理
2. 点击 **"创建作业"**
3. 设置：
   - 作业名称
   - 起止时间
   - 题目列表
   - 评分规则

#### 步骤2：指定学生
- 按班级选择
- 或输入用户名列表
- 生成邀请链接

#### 步骤3：查看完成情况
1. 进入作业详情
2. 查看统计：
   - 完成人数
   - 通过率
   - 每题正确率
3. 导出成绩报表

---

### 2.3 创建比赛

#### 步骤1：比赛设置
1. 点击 **"比赛"** → **"创建比赛"**
2. 配置：
   - 比赛名称
   - 开始/结束时间
   - 比赛类型（OI/IOI/ACM）

#### 步骤2：添加题目
1. 从题库选择，或
2. 上传新题目（zip包）
3. 设置每题分数

#### 步骤3：高级设置
- **罚时**: ACM模式通常为20分钟
- **封榜**: 最后1小时不显示实时排名
- **语言限制**: 允许使用的编程语言
- **可见性**: 公开/私有/邀请制

#### 步骤4：赛前准备
1. 测试所有题目
2. 验证数据正确性
3. 准备题解
4. 通知参赛者

---

### 2.4 题目管理

#### 创建新题目

**步骤1：基本信息**
```
题目ID: p1001
题目名称: 两数之和
tags: [入门, 数学]
difficulty: 入门
```

**步骤2：题目描述**
使用 Markdown 格式：
```markdown
## 题目描述
计算两个整数的和。

## 输入格式
两个整数 a, b

## 输出格式
一个整数，表示 a + b

## 样例
输入：
```
3 5
```

输出：
```
8
```
```

**步骤3：测试数据**
创建数据文件：
```
1.in    1.out
2.in    2.out
3.in    3.out
...
```

打包上传：
```bash
zip data.zip *.in *.out
```

**步骤4：设置限制**
- 时间限制：通常 1s（1000ms）
- 内存限制：通常 256MB
- 输出限制：通常 64MB

### 2.5 Special Judge（SPJ）

当题目有多个正确答案时使用 SPJ。

**示例：浮点数比较**
```cpp
#include <testlib.h>

int main(int argc, char* argv[]) {
    registerTestlibCmd(argc, argv);
    
    double ans = ans.readDouble();
    double out = ouf.readDouble();
    
    if (abs(ans - out) < 1e-6)
        quitf(_ok, "Correct");
    else
        quitf(_wa, "Wrong Answer");
    
    return 0;
}
```

### 2.7 教师常见问题 FAQ

**Q1: 如何防止学生作弊？**
- 开启代码查重
- 设置合理的比赛时间
- 使用原创题目
- IP限制（可选）

**Q2: 如何导入学生名单？**
- 批量注册：上传CSV文件
- 格式：`用户名,邮箱,密码`
- 或生成邀请码让学生自行注册

**Q3: 测试数据如何准备？**
- 边界数据：最小值、最大值
- 随机数据：验证正确性
- 特殊数据：针对易错点
- 通常需要 10-20 组数据

**Q4: 如何设置部分分？**
- 在题目配置中设置 subtask
- 每个 subtask 对应不同分值
- 学生通过部分数据获得部分分数

**Q5: 系统性能如何？**
- 支持 1000+ 人同时在线
- 评测机可横向扩展
- 建议定期检查评测队列
