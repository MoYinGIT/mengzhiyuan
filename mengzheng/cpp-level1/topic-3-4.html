变量

⏱️ 预估学习时长: 55分钟 📋 验证题目: 100题 🎯 难度: 入门级

📚 知识详解

一、什么是变量

变量（Variable）是程序中用于存储数据的容器。变量的值可以在程序运行过程中改变，就像一个有标签的盒子，可以随时更换里面的内容。

💡 一句话理解
变量是程序中存储数据的「容器」，里面的内容可以随时更换。
// 变量示例 int age = 18; // age 是变量，存储整数18 age = 19; // 可以修改为19 age = 20; // 可以再次修改为20 double price = 9.99; // price 是变量，存储价格 price = 8.99; // 价格可以变化

二、变量的三个要素

一个完整的变量包含三个要素：数据类型、变量名、值。

int score = 95; // ↑ ↑ ↑ // 类型 变量名 值
  1. 数据类型：决定变量可以存储什么类型的数据（整数、小数、字符等）
  2. 变量名：标识符，用于引用这个变量（如：age, score, price）
  3. 值：变量实际存储的数据（可以修改）

三、变量的定义和初始化

1. 变量定义

告诉编译器变量的类型和名称，在内存中分配空间。

// 语法：数据类型 变量名; int count; // 定义一个int类型的变量count double price; // 定义一个double类型的变量price char grade; // 定义一个char类型的变量grade
⚠️ 注意
只定义不初始化的变量，其值是不确定的（垃圾值）。使用前务必初始化！

2. 变量初始化

在定义变量的同时给它一个初始值，这是推荐的做法。

// 语法：数据类型 变量名 = 初始值; int age = 18; // 定义并初始化为18 double pi = 3.14159; // 定义并初始化为3.14159 char grade = 'A'; // 定义并初始化为'A' bool passed = true; // 定义并初始化为true

3. 同时定义多个变量

// 同时定义多个同类型变量 int a, b, c; // 定义三个int变量 int x = 1, y = 2, z = 3; // 定义并初始化 // 甚至可以混合 int m = 10, n, p = 20; // m和p初始化，n未初始化

四、变量的赋值和使用

1. 赋值语句

使用等号 = 给变量赋新值。

int score; // 定义变量 score = 85; // 赋值为85 score = 90; // 修改为90 score = score + 5; // 在原有值上加5，变为95
💡 赋值 vs 初始化
• 初始化：定义变量时给值（int a = 10;）
• 赋值：定义后给值（a = 10;）

2. 使用变量

int length = 5; int width = 3; int area = length * width; // 使用变量计算 // 输出变量值 cout << area; // 输出：15

五、变量的生命周期

变量从诞生到消失的整个过程：

1. 定义
分配内存空间
→
2. 初始化
赋予初始值
→
3. 使用

4. 销毁

六、变量的命名规范

✅ 好的命名
studentCount（学生数量）
totalScore（总分）
averagePrice（平均价格）
maxValue（最大值）
❌ 不好的命名
a, b, c（无意义）
x1, x2（不清晰）
shuliang（拼音）
temp（过于笼统）
📋 命名建议
1. 见名知意：从名字就能看出用途
2. 小驼峰命名：studentName, totalCount
3. 避免单字母：除非是循环变量 i, j, k
4. 不用拼音：使用英文单词

七、常见错误

❌ 错误1：使用未初始化的变量
int score; // 未初始化
cout << score; // ❌ 错误：score的值不确定
❌ 错误2：变量类型不匹配
int age = 3.14; // 警告：3.14被截断为3
❌ 错误3：变量名重复定义
int score = 80; // 第一次定义
int score = 90; // ❌ 错误：重复定义
⚠️ 错误4：混淆赋值和比较
if (x = 10) { } // ⚠️ 这是赋值，不是比较！
if (x == 10) { } // ✅ 这才是比较是否相等

「变量是个容器，存数据最行；
类型名和值，三要素记清；
定义再使用，初始化先行；

☯ 容器之道：有之以为利，无之以为用

《道德经》第十一章云：「三十辐共一毂，当其无，有车之用。埏埴以为器，当其无，有器之用……故有之以为利，无之以为用。」

老子以车轮、器皿、房屋为例，说明「有」和「无」的关系：有之以为利，无之以为用。变量的「类型和名」是「有」，为我们提供便利；变量的「可变性、存储空间」是「无」，正是其真正的用途所在。

《道德经》编程对应寓意
三十辐共一毂变量的定义结构框架，支撑整体
当其无，有车之用变量的可变性正因可变，方有其用
埏埴以为器变量的类型塑造形态，容纳数据
有之以为利类型和名称提供便利，便于使用
无之以为用存储和变化真正发挥作用的本质
💡 圣人启示
《论语》云：「工欲善其事，必先利其器。」—— 变量就是我们编程的「器」，正确定义和使用变量，是编写好程序的基础。

变量如器皿，类型定其形，名称便其用，变化尽其能。

变量的存储类型

C++中变量可以指定存储类型，影响其生命周期和作用域：

存储类型说明使用场景
auto自动变量（默认）函数内部的局部变量
static静态变量保留上次调用的值
extern外部变量声明在其他文件中定义的变量
💡 GESP考试提示
GESP一级考试不要求掌握存储类型（static、extern等），只需了解基本概念即可。

变量的作用域

作用域决定了变量在程序的哪些部分可以被访问：

int globalVar = 100; // 全局变量，整个文件可见 void function() { int localVar = 10; // 局部变量，只在函数内可见 // globalVar 在这里也可以用 } int main() { // globalVar 可以在这里使用 // localVar 不能在这里使用（超出作用域） return 0; }

变量交换技巧

交换两个变量的值是常见操作：

// 方法1：使用临时变量（最常见） int temp = a; a = b; b = temp; // 方法2：不使用临时变量（仅限数字） a = a + b; b = a - b; a = a - b; // 方法3：使用C++标准库（GESP一级不要求） // swap(a, b);

常见陷阱

  1. 整数溢出：给int赋值超过范围（如2147483647+1会变成负数）
  2. 浮点精度：float和double有精度限制，不适合精确计算
  3. 类型转换：小类型转大类型安全，大类型转小类型可能丢失数据
  4. 变量覆盖：内层作用域变量会覆盖外层同名变量
📖 学习建议
变量是编程的基础，建议多练习定义、初始化、赋值的操作。养成「定义即初始化」的好习惯，避免使用未初始化的变量。
