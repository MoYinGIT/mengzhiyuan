常量

⏱️ 预估学习时长: 50分钟 📋 验证题目: 100题 🎯 难度: 入门级

📚 知识详解

一、什么是常量

常量（Constant）是程序运行过程中其值不能被改变的量。一旦定义，常量的值就固定不变，就像数学中的π（圆周率）永远是3.14159...一样。

💡 一句话理解
常量就是程序中「一成不变」的值，定义后不能被修改。
// 常量示例 const int MAX_SCORE = 100; // MAX_SCORE 是常量，值固定为100 const double PI = 3.14159; // PI 是常量，值固定为3.14159 const char GRADE = 'A'; // GRADE 是常量，值固定为'A' // 错误：不能修改常量 MAX_SCORE = 200; // ❌ 编译错误：不能修改常量

二、为什么要使用常量

  1. 提高可读性：用有意义的名称代替魔法数字
  2. 便于维护：修改一处，全局生效
  3. 防止误改：编译器会阻止对常量的修改
  4. 表达意图：明确表示某值不应改变
// ❌ 不好的做法：魔法数字 if (score >= 60) { } // 60是什么含义？ int arr[100]; // 100是什么含义？ // ✅ 好的做法：使用常量 const int PASS_SCORE = 60; const int MAX_SIZE = 100; if (score >= PASS_SCORE) { } // 含义清晰 int arr[MAX_SIZE]; // 含义清晰

三、常量的定义方式

方式1：const关键字（推荐）

使用 const 关键字定义常量，这是最常用、最推荐的方式。

const 数据类型 常量名 = 初始值; // 示例 const int MAX_STUDENTS = 50; const double PI = 3.1415926; const char NEWLINE = '\n'; const bool DEBUG = true;

方式2：#define宏定义

使用预处理器指令 #define 定义常量，在编译前进行文本替换。

#define 常量名 值 // 示例 #define MAX_SIZE 100 #define PI 3.14159 #define GREETING "Hello, World!" // 使用时 int arr[MAX_SIZE]; // 编译时替换为 int arr[100];
⚠️ const vs #define
• const 有类型检查，更安全
• #define 只是文本替换，无类型检查
• 推荐优先使用 const

方式3：直接量（字面常量）

直接写在代码中的常量值，如 100、3.14、'A' 等。

int a = 100; // 100 是整数常量 double b = 3.14; // 3.14 是浮点常量 char c = 'X'; // 'X' 是字符常量

四、常量的类型

🔢 整数常量

如：100、-50、0

后缀：100L(长整型)、100U(无符号)

🔢 浮点常量

如：3.14、-0.5、2.5e3

后缀：3.14f(float)、3.14(double)

🔤 字符常量

用单引号包围：'A'、'\n'

转义字符：'\n'(换行)、'\t'(制表符)

📜 字符串常量

用双引号包围："Hello"

实际上是字符数组，末尾自动加'\0'

✅ 布尔常量

只有两个：true 和 false

分别表示真和假

五、常量 vs 变量

🔵 变量

  • 值可以改变
  • 用普通声明
  • 存储在内存中
  • 如：int age = 18;

🟡 常量

  • 值不能改变
  • 用 const 声明
  • 也存储在内存中
  • 如：const int MAX = 100;
// 变量：值可以改变 int score = 80; score = 90; // ✅ 可以修改 score = 100; // ✅ 可以再次修改 // 常量：值不能改变 const int PASS = 60; // PASS = 70; // ❌ 错误：不能修改常量

六、常见错误

❌ 错误1：修改常量
const int MAX = 100;
MAX = 200; // 编译错误：不能修改常量
❌ 错误2：常量未初始化
const int MAX; // 错误：常量必须初始化
MAX = 100;
⚠️ 错误3：混淆赋值与初始化
const int MAX = 100; // ✅ 初始化（定义时赋值）
const int MAX; MAX = 100; // ❌ 错误：不能在定义后赋值

七、GESP考试要点

  • 常量的概念：程序运行过程中值不能被改变的量
  • 常量的定义：const 关键字的使用方法
  • 常量的类型：整数、浮点、字符、字符串、布尔常量
  • 常量vs变量：能区分常量和变量的区别
  • 常见错误：常量必须初始化、不能修改常量
🎯 核心口诀
「常量值不变，const来定义；
定义即初始化，之后不能改；
整浮字符布尔，类型要记牢。」

☯ 常变之道：不易、变易、简易

《易经》云：「易有太极，是生两仪。...易有三义：不易、变易、简易。」

《易经》之「易」有三重含义：不易（不变）、变易（变化）、简易（简单）。常量对应「不易」，变量对应「变易」，良好的程序设计则是「简易」。

易经三义编程对应寓意
不易（不变）常量 const根本规律，恒定不变
变易（变化）变量 var灵活应变，适应需求
简易（简单）良好设计化繁为简，大道至简
💡 圣人启示
《道德经》云：「不知常，妄作凶。」—— 不了解不变的规律，轻举妄动就会出乱子。

在编程中，将不应改变的值设为常量（知常），可以避免误改导致的错误（不妄作）。

constexpr：编译期常量（C++11）

C++11 引入了 constexpr 关键字，用于定义编译期常量。与 const 相比：

  • const：运行期常量，值在运行时确定
  • constexpr：编译期常量，值在编译时确定
const int size1 = getSize(); // ✅ const 可以运行时初始化 constexpr int size2 = 100; // ✅ constexpr 必须编译期确定 // constexpr int size3 = getSize(); // ❌ 错误：getSize() 不是编译期函数
💡 GESP考试提示
GESP一级考试只考察 const 关键字，constexpr 属于C++11新特性，一级不要求掌握。

特性#defineconst

常见的编程常量约定

  1. 全大写下划线命名：常量名通常全部大写，单词间用下划线分隔
    如：MAX_SIZE、PI、PASS_SCORE
  2. 魔法数字：避免在代码中直接使用没有解释的数字，应该用常量替代
  3. 一处修改：使用常量的好处是修改一处，到处生效
📖 学习建议
养成「见数就疑」的习惯：看到代码中的数字，先想想是否应该定义为常量。这不仅让代码更易读，也更容易维护。
