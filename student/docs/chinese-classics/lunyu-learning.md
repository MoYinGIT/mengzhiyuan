## 1️⃣ 学而不思则罔

### 原文

> "学而不思则罔，思而不学则殆。" ——《论语·为政》

**译文**：
只学习不思考，就会迷惑；只思考不学习，就会危险。

### 编程中的应用

**❌ 学而不思**（复制粘贴型学习）：
```cpp
// 网上抄了一段排序代码
// 不理解原理，只是能运行
sort(arr, arr + n);  // 这是啥？不懂，反正能排序
```

**❌ 思而不学**（空想型学习）：
```
"我要自己发明一个排序算法！"
（不看经典算法，闭门造车，最后发现是冒泡排序的变种）
```

**✅ 学思结合**（正确姿势）：
```cpp
// 1. 学习经典算法（学）
void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
            }
        }
    }
}

// 2. 思考为什么这样写（思）
// - 为什么外层循环n-1次？
// - 为什么内层循环n-1-i次？
// - 能不能优化？

// 3. 尝试改进（学思结合）
void bubble_sort_optimized(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break;  // 已经有序，提前退出
    }
}
```

### 学习方法：三步法

```
【学】看懂代码
    ↓
【思】理解原理
    ↓
【行】动手实践
    ↓
【证】验证理解
```

## 2️⃣ 教学相长

### 原文

> "学然后知不足，教然后知困。" ——《礼记·学记》

### 编程中的应用

**教是最好的学**：

| 阶段 | 行为 | 收获 |
|:---|:---|:---|
| 学 | 看书、听课 | 知道知识 |
| 做 | 写代码 | 掌握技能 |
| 教 | 给别人讲解 | 发现盲点 |
| 写 | 写博客、文档 | 系统梳理 |

**实践建议**：
1. **做笔记** - 用自己的话记录
2. **写博客** - 分享学习心得
3. **教同学** - 解答别人的问题
4. **做视频** - 录制教程

**代码审查（Code Review）**：
```
别人审查你的代码：
- 发现bug
- 提出改进建议
- 学习更好的写法

你审查别人的代码：
- 学习不同思路
- 发现共性问题
- 提升代码品味
```

## 3️⃣ 温故而知新

### 原文

> "温故而知新，可以为师矣。" ——《论语·为政》

### 编程中的应用

**复习的重要性**：

```
第1次学：理解概念        → 记忆度 40%
第2次看：加深印象        → 记忆度 60%
第3次用：实践应用        → 记忆度 80%
第4次教：讲解给别人      → 记忆度 95%
```

**复习策略**：

1. **当天复习** - 学完当天回顾
2. **隔天复习** - 第二天再动手写一遍
3. **周复习** - 周末总结本周所学
4. **月复习** - 每月回顾，建立知识体系

**示例：错题本**

```cpp
/* 
 * 【错题记录】
 * 日期：2026-03-23
 * 题目：数组越界
 * 
 * 错误代码：
 * int arr[5] = {1, 2, 3, 4, 5};
 * cout << arr[5];  // 错误！索引从0到4
 * 
 * 正确代码：
 * cout << arr[4];  // 最后一个元素
 * 
 * 教训：
 * - 数组索引从0开始
 * - 最后一个元素的索引是 n-1
 * - 循环条件注意 i < n，不是 i <= n
 */
```

## 4️⃣ 知之为知之

### 原文

> "知之为知之，不知为不知，是知也。" ——《论语·为政》

### 编程中的应用

**承认不知道**：

❌ **不懂装懂**：
```
"这个Bug我知道怎么解"
（其实不知道，乱改一通，问题更严重）
```

✅ **诚实面对**：
```
"这个错误我没见过，让我查一下资料"
（查文档、搜索、问人，真正解决问题）
```

**提问的艺术**：

| 不好的提问 | 好的提问 |
|:---|:---|
| "这个怎么错了？" | "第5行报错'syntax error'，是哪里语法不对？" |
| "帮我看看" | "我尝试了A方法，结果出现B错误，为什么？" |
| "怎么写排序？" | "冒泡排序和选择排序哪个更适合这个场景？" |

**学会搜索**：
1. **读报错** - 错误信息是线索
2. **查文档** - 官方文档最权威
3. **搜问题** - 用英文搜索技术问题
4. **问社区** - Stack Overflow、GitHub Issues

## 5️⃣ 三人行必有我师

### 原文

> "三人行，必有我师焉。择其善者而从之，其不善者而改之。" ——《论语·述而》

### 编程中的应用

**向他人学习**：

**学习优秀代码**：
```cpp
// 看到别人写的好代码
// 学习它的：
// - 命名规范
// - 代码结构
// - 注释风格
// - 算法选择
```

**从错误中学习**：
```cpp
// 看到别人犯的错误
// 提醒自己：
// - 不要犯同样的错
// - 理解为什么会错
// - 思考如何避免
```

**开源社区学习**：
- **阅读源码** - Linux、Redis、Vue等优秀项目
- **参与贡献** - 从改文档开始
- **交流讨论** - 技术社区、会议

## 7️⃣ 持之以恒

### 原文

> "譬如为山，未成一篑，止，吾止也。譬如平地，虽覆一篑，进，吾往也。" ——《论语·子罕》

**译文**：
好比堆土成山，只差一筐土就完成了，停下来，是我自己要停的。好比平地堆山，虽然只倒了一筐土，继续下去，也是我自己要去做的。

### 编程学习的心态

**不要半途而废**：
```
第1周：充满热情
第2周：遇到困难
第3周：想放弃（关键！）
第4周：度过难关
第5周：继续前进
...
第1年：成为高手
```

**每日进步**：
```
每天进步1%，一年进步37倍
每天退步1%，一年只剩3%

数学表达：
(1 + 0.01)^365 ≈ 37.78
(1 - 0.01)^365 ≈ 0.03
```
