## 📚 知识探索

### 🔍 探索1：循环——让代码"重复做"

**生活类比**：
- 跳绳：跳一下、再跳一下...直到跳满100下
- 刷牙：左边刷、右边刷、上面刷...持续2分钟
- 跑步：左脚、右脚、左脚、右脚...一直跑

**什么是循环？**
> 把要做的事情写一遍，让计算机重复执行多次。

**Scratch的三种循环积木**：

| 积木 | 作用 | 生活例子 |
|:---|:---|:---|
| 重复执行 | 一直重复，永不停止 ⏵ | 心脏一直跳动 |
| 重复执行 (10) 次 | 重复固定次数 | 跳绳100下 |
| 重复执行直到 <条件> | 直到满足条件才停止 | 一直吃直到吃饱 |

**试一试**：让小猫走正方形

**不用循环的写法**（要写很多遍）：
```scratch
移动 100 步
右转 90 度
移动 100 步
右转 90 度
移动 100 步
右转 90 度
移动 100 步
右转 90 度
```

**用循环的写法**（只要写一遍）：
```scratch
重复执行 (4) 次
    移动 (100) 步
    右转 (90) 度
结束
```

**发现规律了吗？** 🤔
- 正方形的4条边都一样长
- 每次转的角度也一样（90度）
- 所以重复4次就画出了正方形！

### 🔍 探索2：变量——程序中的"盒子"

**生活类比**：
想象你有几个盒子，上面贴着标签：
- 📦 「分数」盒子 —— 用来装游戏分数
- 📦 「生命值」盒子 —— 用来装还剩几条命
- 📦 「时间」盒子 —— 用来装倒计时秒数

**什么是变量？**
> 一个带名字的"盒子"，可以存数字或文字，里面的内容可以改变。

**创建变量的步骤**：
1. 点击「变量」分类
2. 点击「建立一个变量」
3. 输入变量名字（比如"分数"）
4. 点击确定

**变量的基本操作**：

| 积木 | 作用 |
|:---|:---|
| 将 [分数] 设为 (0) | 把盒子清空，放入0 |
| 将 [分数] 增加 (1) | 往盒子里加1 |
| 显示变量 [分数] | 在舞台上显示这个变量 |
| 隐藏变量 [分数] | 不在舞台上显示 |

**试一试**：创建一个计分器

```scratch
当绿旗被点击
将 [分数 v] 设为 (0)        // 游戏开始，分数归零
重复执行
    等待 (1) 秒
    将 [分数 v] 增加 (1)     // 每过1秒，分数加1
结束
```

### 🔍 探索3：条件判断——程序的"岔路口"

**生活场景**：
每天出门前，你都会做判断：
- 如果下雨 → 带伞
- 如果不下雨 → 不带伞

**这就是条件判断！**

**Scratch的条件积木**：

#### 形式1：如果...那么（二选一）
```scratch
如果 <条件> 那么
    // 条件为真时执行
结束
```

#### 形式2：如果...那么...否则（二选二）
```scratch
如果 <条件> 那么
    // 条件为真时执行
否则
    // 条件为假时执行
结束
```

**常用的条件积木**（六边形）：

| 条件 | 含义 |
|:---|:---|
| < ( ) > ( ) > | 左边大于右边？ |
| < ( ) < ( ) > | 左边小于右边？ |
| < ( ) = ( ) > | 两边相等？ |
| <碰到 [边缘 v]？> | 碰到边缘了吗？ |
| <碰到颜色 (#FF0000)？> | 碰到红色了吗？ |
| <按下 [空格 v] 键？> | 按空格了吗？ |

**试一试**：小猫在哪里？

```scratch
当绿旗被点击
重复执行
    如果 <(x坐标) > (0)> 那么
        说 [我在右边！] 0.5秒
    否则
        说 [我在左边！] 0.5秒
    结束
    移动 (5) 步
    等待 (0.5) 秒
结束
```

**发现规律了吗？**
- 小猫在右边（x>0）时说"我在右边"
- 小猫在左边（x≤0）时说"我在左边"
- 程序会自动判断，每0.5秒检查一次！

### 🔍 探索4：嵌套结构——积木套积木

**什么是嵌套？**
> 把一种积木放到另一种积木里面，就像俄罗斯套娃。

**例子**：重复执行里面放如果
```scratch
重复执行
    如果 <碰到颜色 (#FF0000)？> 那么
        说 [我碰到红色了！] 2秒
    结束
结束
```

**解读**：
- 外层的「重复执行」让程序一直运行
- 内层的「如果」检查是否碰到红色
- 只有碰到红色时才会说话

**想一想**：下面这段代码会做什么？

```scratch
重复执行 (5) 次
    移动 (20) 步
    如果 <碰到边缘？> 那么
        碰到边缘就反弹
    结束
结束
```

<details>
<summary>✅ 答案</summary>

**解释**：
- 小猫会向前移动5次，每次20步
- 每次移动后检查是否碰到边缘
- 如果碰到了，就反弹（转向）
- 这样小猫就不会跑出舞台了！

</details>

### 🟡 练习3：倒计时器（进阶）

**要求**：制作一个10秒倒计时器，时间到后说"时间到！"

**分解步骤**：
- Step 1：创建变量「时间」
- Step 2：设为10
- Step 3：重复执行，每次减1，直到时间为0
- Step 4：时间到后说"时间到！"

<details>
<summary>✅ 参考答案</summary>

```scratch
当绿旗被点击
将 [时间 v] 设为 (10)
重复执行直到 <(时间) = (0)>
    说 (时间) 1秒
    等待 (1) 秒
    将 [时间 v] 增加 (-1)
结束
说 [时间到！] 2秒
```

**解释**：
- 「增加 (-1)」就是减1
- 「重复执行直到」会一直循环，直到时间等于0才停止

</details>

### 🐛 常见错误自查

| 错误 | 原因 | 解决 |
|:---|:---|:---|
| 循环不停止 | 用了「重复执行」忘记加停止条件 | 改成「重复执行N次」或加条件判断 |
| 变量不变化 | 忘记使用「将变量增加」 | 检查是否用了正确的积木 |
| 条件不生效 | 条件写反了（比如该用>用了<） | 检查比较符号 |
| 角色检测不到碰撞 | 角色名拼写错误 | 检查角色名称是否匹配 |
| 嵌套积木混乱 | 积木没有正确套进去 | 拖动时看白色高亮提示 |
