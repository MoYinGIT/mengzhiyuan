## 📚 知识探索

### 🔍 探索1：列表——变量的"升级版"

**生活类比**：
- **变量** = 一个盒子，装一个东西
- **列表** = 一排盒子，可以装很多东西，每个盒子有编号

**例子**：
- 变量「当前分数」= 只存一个数字
- 列表「历史分数」= 可以存很多分数：[100, 150, 80, 200, 120]

**创建列表的步骤**：
1. 点击「变量」分类
2. 点击「建立一个列表」
3. 输入列表名称（比如"购物清单"）
4. 选择"适用于所有角色"或"仅适用于当前角色"
5. 点击确定

**列表的基本操作**：

| 积木 | 作用 |
|:---|:---|
| 将 [东西] 加入 [购物清单] | 在列表末尾添加一项 |
| 删除 [购物清单] 的第 (1) 项 | 删除指定位置的内容 |
| 将 [购物清单] 的第 (1) 项替换为 [东西] | 修改指定位置的内容 |
| 清空 [购物清单] | 删除所有内容 |
|  shopping清单 的第 (1) 项 | 获取指定位置的内容 |
| [购物清单] 的项目数 | 列表里有几项 |

**试一试**：制作一个购物清单

```scratch
当绿旗被点击
删除 [购物清单 v] 的全部项目    // 清空列表
将 [苹果] 加入 [购物清单 v]     // 加入苹果
将 [香蕉] 加入 [购物清单 v]     // 加入香蕉
将 [牛奶] 加入 [购物清单 v]     // 加入牛奶
说 (购物清单) 2秒                // 说出整个列表
```

**发现规律了吗？** 🤔
- 列表可以存很多个东西
- 每个东西都有编号（第1项、第2项...）
- 可以用「说 (列表名)」显示整个列表内容

### 🔍 探索2：遍历列表——逐个查看

**什么是遍历？**
> 一个一个地查看列表里的每一项。

**生活例子**：点名册上有全班同学的名字，老师要逐个叫名字，这就是"遍历"。

**遍历列表的方法**：

```scratch
// 方法：用变量i作为"计数器"
当绿旗被点击
将 [i v] 设为 (1)                    // 从第1项开始
重复执行 ([购物清单 v] 的项目数) 次    // 重复次数 = 列表长度
    说 (购物清单 的第 (i) 项) 2秒     // 说出第i项
    将 [i v] 增加 (1)                 // i加1，看下一项
结束
```

**发现规律了吗？**
- 变量i从1开始，每次加1
- 第一次说"第1项"，第二次说"第2项"...
- 直到说完所有项

### 🔍 探索3：自定义积木——自己造积木

**为什么要自定义积木？**

想象你要画很多个正方形：
```scratch
// 第一个正方形
重复执行 (4) 次
    移动 (50) 步
    右转 (90) 度
结束

// 第二个正方形（代码一模一样！）
重复执行 (4) 次
    移动 (50) 步
    右转 (90) 度
结束

// 太麻烦了！
```

**自定义积木的好处**：
- 把常用代码"打包"成一个积木
- 需要时直接拖出来用
- 代码更简洁，更容易修改

**创建自定义积木的步骤**：
1. 点击「我的积木」分类
2. 点击「制作新的积木」
3. 输入积木名称（比如"画正方形"）
4. 点击确定
5. 定义积木的代码（就像写普通程序）
6. 使用时从「我的积木」中拖出来

**试一试**：创建一个"画正方形"积木

```scratch
定义 画正方形
重复执行 (4) 次
    移动 (50) 步
    右转 (90) 度
结束
```

**使用自定义积木**：
```scratch
当绿旗被点击
落笔
画正方形          // 直接拖出来用！
移动 (100) 步
画正方形          // 再画一个！
画正方形          // 又画一个！
抬笔
```

### 🔍 探索4：带参数的自定义积木

**什么是参数？**

之前的"画正方形"只能画固定大小（50步）。如果想画不同大小的正方形呢？

**带参数的积木**：
```scratch
定义 画正方形 边长
重复执行 (4) 次
    移动 (边长) 步      // 使用参数！
    右转 (90) 度
结束
```

**使用带参数的积木**：
```scratch
当绿旗被点击
落笔
画正方形 (50)      // 画边长50的正方形
画正方形 (100)     // 画边长100的正方形
画正方形 (30)      // 画边长30的正方形
抬笔
```

**发现规律了吗？**
- 参数让积木更灵活
- 同一个积木可以做不同的事情
- 就像函数的"输入值"

### 🟡 练习4：随机抽题器（进阶）

**要求**：制作一个"问答游戏"：
1. 创建一个列表"题目"，里面有5道数学题
2. 创建一个列表"答案"，对应每道题的答案
3. 点击绿旗，随机出一道题
4. 玩家输入答案后，判断对错

**Step分解**：
- Step 1：创建两个列表，填入题目和答案
- Step 2：生成随机数（1到5），选出题目编号
- Step 3：说出对应的题目
- Step 4：询问玩家答案
- Step 5：判断答案是否正确

<details>
<summary>💡 需要提示？</summary>

**提示**：
1. 创建列表"题目"，加入："1+1=?"、"2+3=?"...等
2. 创建列表"答案"，加入："2"、"5"...等
3. 用「随机数 (1) 到 (5)」选题目
4. 用「题目 的第 (编号) 项」获取题目内容
5. 用「询问...并等待」获取玩家输入
</details>

<details>
<summary>✅ 参考答案</summary>

**初始化（绿旗点击）**：
```scratch
当绿旗被点击
删除 [题目 v] 的全部项目
删除 [答案 v] 的全部项目
将 [1+1=?] 加入 [题目 v]
将 [2] 加入 [答案 v]
将 [2+3=?] 加入 [题目 v]
将 [5] 加入 [答案 v]
将 [10-4=?] 加入 [题目 v]
将 [6] 加入 [答案 v]
将 [3×2=?] 加入 [题目 v]
将 [6] 加入 [答案 v]
将 [8÷2=?] 加入 [题目 v]
将 [4] 加入 [答案 v]
```

**出题和判断**：
```scratch
当按下 [空格 v] 键
将 [题号 v] 设为 (随机数 (1) 到 (5))
说 (题目 的第 (题号) 项) 2秒
询问 [你的答案是？] 并等待
如果 <(回答) = (答案 的第 (题号) 项)> 那么
    说 [答对了！真棒！] 2秒
否则
    说 [答错了，再想想！] 2秒
结束
```

</details>

### 🔴 练习6：完整游戏设计（挑战）

**游戏名称**：小猫躲陨石

**游戏规则**：
1. 小猫在屏幕底部，可以用左右键移动
2. 陨石从屏幕上方随机位置掉落
3. 陨石碰到小猫，游戏结束
4. 躲避一颗陨石得1分
5. 随着时间推移，陨石掉落速度越来越快

**需要的元素**：
- 变量：分数、游戏状态（是否结束）、陨石速度
- 角色：小猫（玩家）、陨石（多个克隆）
- 控制：左右键移动
- 机制：克隆、碰撞检测、难度递增

**想一想**：怎么让陨石越来越多、越来越快？

<details>
<summary>💡 需要提示？</summary>

**关键设计**：
1. 陨石使用「克隆」功能，一个角色变多个
2. 每隔几秒克隆一个陨石
3. 陨石速度用变量控制，随分数增加
4. 游戏结束用变量控制，停止生成陨石

**核心代码思路**：
- 小猫：左右键控制 + 检测碰撞
- 陨石本体：每隔N秒克隆自己
- 陨石克隆体：从上往下掉，到底部删除
- 主程序：初始化变量，循环生成陨石
</details>

### 🐛 常见错误自查

| 错误 | 原因 | 解决 |
|:---|:---|:---|
| 列表为空 | 忘记初始化加入数据 | 用「将...加入...」先加数据 |
| 列表越界 | 访问了不存在的项（比如第6项，但只有5项） | 检查索引是否在1到项目数之间 |
| 自定义积木没反应 | 没有调用积木，只定义了 | 需要把积木拖到程序中使用 |
| 参数不生效 | 参数名和使用的变量名不一致 | 检查拼写是否相同 |
| 克隆体不消失 | 忘记删除克隆体 | 在代码末尾加「删除此克隆体」 |
