## 📚 知识探索

### 🔍 探索1：为什么需要排序？

**场景**：手机通讯录是按什么顺序排列的？

**想一想**：如果通讯录不排序，找一个人要多久？

**答案**：
- 不排序：可能要翻遍整个通讯录（O(n)）
- 排序后：可以用二分查找（O(log n)）

**排序的应用**：
- 排行榜（游戏积分排名）
- 搜索结果（按相关性排序）
- 数据统计（找中位数需要先排序）

---

### 🔍 探索2：冒泡排序——相邻交换

**核心思想**：相邻元素两两比较，大的往后"冒泡"

**手动模拟**：对 [5, 3, 8, 6, 2] 排序

**第1轮**：
```
[5, 3, 8, 6, 2]  5和3比较，交换 → [3, 5, 8, 6, 2]
[3, 5, 8, 6, 2]  5和8比较，不交换
[3, 5, 8, 6, 2]  8和6比较，交换 → [3, 5, 6, 8, 2]
[3, 5, 6, 8, 2]  8和2比较，交换 → [3, 5, 6, 2, 8]
```
**结果**：最大的8"冒"到了最后！

**第2轮**：对前4个 [3, 5, 6, 2] 重复...

**发现规律了吗？**
- 每轮确定一个数的位置（最大的沉底）
- n个数需要n-1轮
- 第i轮只需要比较n-i次

```cpp
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {           // n-1轮
        for (int j = 0; j < n-1-i; j++) {     // 每轮少比较一次
            if (arr[j] > arr[j+1]) {           // 相邻比较
                int temp = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = temp;
            }
        }
    }
}
```

### 🔍 探索3：二分查找——猜数字游戏

**场景**：猜1-100中的一个数，怎么猜最快？

❌ **笨方法**：从1开始猜（最坏猜100次）

✅ **聪明方法**：二分查找（最多7次！）

**策略**：
1. 猜中间数 50
2. 如果"大了"，在1-49中猜
3. 如果"小了"，在51-100中猜
4. 重复直到猜中

**为什么快？**
- 每次排除一半的可能性！
- 100 → 50 → 25 → 13 → 7 → 4 → 2 → 1

---

<details>
<summary>✅ 答案对照</summary>

```cpp
int binarySearch(int arr[], int n, int target) {
    int left = 0;
    int right = n - 1;          // 右边界是n-1
    
    while (left <= right) {
        int mid = (left + right) / 2;   // 除以2
        
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) left = mid + 1;   // 在右半边
        else right = mid - 1;                          // 在左半边
    }
    return -1;
}
```
</details>

### 🔍 探索4：算法效率——时间复杂度

**问题**：冒泡排序和二分查找，谁更快？

**不是比较一次操作，而是比较增长趋势**：

| 算法 | 时间复杂度 | 说明 |
|:---|:---|:---|
| 冒泡排序 | O(n²) | n个数比较约n²次 |
| 二分查找 | O(log n) | 每次排除一半 |
| 线性查找 | O(n) | 一个一个找 |

**实际对比**：

| 数据量 | O(n) | O(n²) | O(log n) |
|:---:|:---:|:---:|:---:|
| 10 | 10 | 100 | 4 |
| 1000 | 1000 | 1,000,000 | 10 |
| 100万 | 100万 | 1万亿 | 20 |

**结论**：算法的选择很重要！

## ✍️ 动手实践

### 🟢 练习1：选择排序（基础）

**选择排序思想**：
1. 找最小元素，和第1个交换
2. 在剩余中找最小，和第2个交换
3. 重复直到有序

**手动模拟**：[5, 3, 8, 6, 2]
```
[5, 3, 8, 6, 2]  最小是2，和5交换 → [2, 3, 8, 6, 5]
[2, 3, 8, 6, 5]  剩余最小3，已在位置 → [2, 3, 8, 6, 5]
[2, 3, 8, 6, 5]  剩余最小5，和8交换 → [2, 3, 5, 6, 8]
...
```

**试一试**：写出选择排序代码

<details>
<summary>✅ 参考答案</summary>

```cpp
void selectionSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        int minIndex = i;
        // 找最小值的下标
        for (int j = i+1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        // 交换
        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }
}
```
</details>

### 🟡 练习2：STL sort使用（进阶）

**实际竞赛中**，手写排序太慢，用STL：

```cpp
#include <algorithm>  // 需要包含头文件

int main() {
    int arr[5] = {5, 3, 8, 6, 2};
    
    // 排序
    sort(arr, arr + 5);  // 从arr[0]到arr[4]排序
    
    // 输出
    for (int i = 0; i < 5; i++) {
        cout << arr[i] << " ";
    }
    return 0;
}
```

**从大到小排序**：
```cpp
sort(arr, arr + 5, greater<int>());
```

**结构体排序**：
```cpp
struct Student {
    string name;
    int score;
};

// 按成绩排序
bool cmp(Student a, Student b) {
    return a.score > b.score;  // 成绩高的在前
}

sort(stu, stu + n, cmp);
```

### 🔴 练习3：快速排序思想（挑战）

**快排核心**：选一个"基准"，小的放左边，大的放右边

```
[6, 3, 8, 2, 9, 1]  选6作为基准
      ↓
[3, 2, 1]  [6]  [8, 9]  
  <6      基准   >6
      ↓
左边排序 + 基准 + 右边排序
```

**思考题**：快排为什么快？

<details>
<summary>💡 提示</summary>

快排平均时间复杂度O(n log n)，比冒泡O(n²)快得多！
</details>

### 🐛 常见错误自查

| 错误 | 现象 | 解决 |
|:---|:---|:---|
| 二分查找边界错误 | 死循环或漏查 | 记住left=mid+1, right=mid-1 |
| 冒泡排序越界 | 数组越界 | j < n-1-i |
| 忘记数组要先排序 | 二分查找失败 | 二分只能用于有序数组 |

## 📚 知识速查表

| 算法 | 思想 | 时间复杂度 | 代码 |
|:---|:---|:---:|:---|
| 冒泡排序 | 相邻交换 | O(n²) | 双重循环 |
| 选择排序 | 选最小交换 | O(n²) | 找最小再交换 |
| 快速排序 | 基准分治 | O(n log n) | 递归 |
| 二分查找 | 折半查找 | O(log n) | while循环 |
| STL sort | 底层优化 | O(n log n) | `sort(arr, arr+n)` |

**算法选择指南**：
- 数据量小（n < 1000）：任何排序都行
- 数据量大：用STL sort
- 查找无序数组：线性查找
- 查找有序数组：二分查找
