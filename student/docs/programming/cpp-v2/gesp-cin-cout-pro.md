## 【大题1】cin的"分隔符陷阱"

**考察点**：cin >> 以空白字符（空格、换行、制表符）为分隔符

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string a, b, c;
    cin >> a >> b >> c;
    cout << "[" << a << "]" << endl;
    cout << "[" << b << "]" << endl;
    cout << "[" << c << "]" << endl;
    return 0;
}
```

**输入**：`Hello World C++ Programming`

### 一、选择题

**1. 程序第一行输出是（ ）**

A. [Hello]  
B. [Hello World]  
C. [HelloWorld]  
D. [Hello World C++]

**2. 程序第二行输出是（ ）**

A. [World C++]  
B. [World]  
C. [C++]  
D. [WorldC++]

**3. 如果要读取包含空格的完整一行，应该使用（ ）**

A. cin >> line  
B. cin.get(line)  
C. getline(cin, line)  
D. scanf("%s", line)

### 二、判断题

**4. `cin >> a >> b >> c` 会读取三个以空格分隔的字符串。**  
（ ）正确 （ ）错误

**5. cin 读取字符串时，遇到空格会停止读取，空格会被消耗掉（不保留在缓冲区）。**  
（ ）正确 （ ）错误

**6. 上面的程序读取后，"Programming"还留在输入缓冲区中，未被读取。**  
（ ）正确 （ ）错误

### 三、填空题

**7. 如果输入是 `abc def`，变量a的值是 ______，变量b的值是 ______。**

**8. 如果只想读取两个单词，但输入了三个，第三个单词会 ______（保留/消失）在输入缓冲区。**

**9. cin读取时，默认以 ______（空格/换行/制表符/以上全部）作为分隔符。**

---

**解析关键点**：
- cin >> 以任意空白字符为分隔符
- 空格、换行、制表符都会分隔输入项
- 分隔符会被消耗，不会读入变量
- 多余输入留在缓冲区


Hello World`

**1. 程序运行后，line的内容是（ ）**

A. Hello World  
B. （空字符串）

A. getline读取失败了  
B. cin >> n后，换行符\n留在缓冲区，被getline读走  
C. line变量未初始化  
D. 程序有语法错误

**3. 如何修复这个问题（ ）**

A. 在getline前加 cin.clear()  
B. 在getline前加 cin.ignore() 消耗掉换行符  
C. 把getline换成cin >> line  
D. 重启程序

### 二、判断题

**4. `cin >> n` 读取数字后，会留下输入结束时的换行符\n在缓冲区。**  
（ ）正确 （ ）错误

**5. `getline(cin, line)` 会读取整行，包括换行符，但存储时不包含换行符。**  
（ ）正确 （ ）错误

**6. cin >> 和 getline 混用时，通常需要在中间加 cin.ignore() 来消耗换行符。**  
（ ）正确 （ ）错误

### 三、填空题

**7. cin >> 读取数字后，缓冲区中还留有一个 ______ 字符。**

**8. 解决方法：在getline之前调用 `cin.______()` 来消耗残留的换行符。**

**9. 如果连续调用两次getline，输入两行文字，第二次getline读取的是第 ______ 行。**

**解析关键点**：
- cin >> n 读取42，留下\n在缓冲区

## 【大题4】cout的"格式化精度陷阱"

**考察点**：setprecision 与 fixed 的区别

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double pi = 3.1415926535;
    
    cout << setprecision(4) << pi << endl;           // 输出1
    cout << fixed << setprecision(4) << pi << endl;  // 输出2
    cout << pi << endl;                              // 输出3
    
    return 0;
}
```

### 一、选择题

**1. 输出1的结果是（ ）**

A. 3.1415  
B. 3.142  
C. 3.1416  
D. 3.1415926535

**2. 输出2的结果是（ ）**

A. 3.1415  
B. 3.1416  
C. 3.142  
D. 3.1415926535

**3. 输出3的结果是（ ）**

A. 3.1415926535  
B. 3.1416  
C. 3.142  
D. 与输出2相同

### 二、判断题

**4. `setprecision(4)` 单独使用时，控制的是有效数字位数（总共4位）。**  
（ ）正确 （ ）错误

**5. `fixed << setprecision(4)` 控制的是小数点后的位数（固定4位小数）。**  
（ ）正确 （ ）错误

**6. 一旦使用了fixed，后续的输出都会保持fixed格式，直到程序结束。**  
（ ）正确 （ ）错误

### 三、填空题

**7. 要输出3.1416（保留4位小数），应该使用 `cout << ______ << setprecision(4) << pi;`**

**8. 要输出3.142（保留4位有效数字），应该使用 `cout << setprecision(______)` 而不加fixed。**

**9. 要取消fixed格式，恢复到默认格式，可以使用 `cout << ______;`**

---

**解析关键点**：
- setprecision(4)：有效数字4位 → 3.142
- fixed + setprecision(4)：小数点后4位 → 3.1416
- fixed状态持续，需要用defaultfloat取消

## 【大题10】endl与'\n'的"性能陷阱"

**2. endl 和 '\n' 的主要区别是（ ）**

A. 没有区别  
B. endl会换行，'\n'不会  
C. endl会换行并刷新缓冲区，'\n'只换行  
D. endl是C++的，'\n'是C的

**3. 在什么情况下应该使用endl而不是'\n'（ ）**

## 教学建议

### 实验验证
建议学生实际运行这些代码，观察结果，加深理解：

```cpp
// 实验1：观察分隔符
string a, b;
cin >> a >> b;  // 输入 "Hello World"
cout << "[" << a << "][" << b << "]" << endl;

// 实验2：观察换行符残留
int n; string line;
cin >> n; 
getline(cin, line);  // 读取到空行
cout << "n=" << n << ", line=[" << line << "]" << endl;

// 实验3：观察类型不匹配
int x; 
cin >> x;  // 输入 "abc"
cout << "cin状态: " << cin.fail() << ", x=" << x << endl;
```

### 核心要点总结

1. **cin >>**：以空白分隔，跳过空白，遇到不匹配停止
2. **getline**：读取整行，包括空格，注意换行符残留
3. **cin >> char**：跳过空白，cin.get()：不跳过空白
4. **setprecision**：默认有效数字，fixed后是小数位
5. **setw**：只生效一次，默认右对齐
6. **endl vs \n**：endl刷新缓冲区，性能更低
7. **输入失败**：设置fail标志，需clear+ignore恢复
