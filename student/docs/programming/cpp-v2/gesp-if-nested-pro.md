## 核心要点总结

### if嵌套使用检查清单

- [ ] 理解嵌套的基本结构：if块内再包含if
- [ ] 外层条件满足才会判断内层条件
- [ ] 每增加一层嵌套，代码缩进一级
- [ ] 嵌套层级建议不超过3层
- [ ] `if (a) { if (b) }` 等价于 `if (a && b)`，后者更简洁
- [ ] 内层定义的变量只在内层有效
- [ ] else与最近的未匹配if匹配，用花括号明确配对
- [ ] 可以通过提前返回或使用逻辑运算符优化嵌套
- [ ] 保持逻辑等价的前提下减少嵌套层级

### 常见错误对比

| 错误 | 问题 | 正确做法 |
|:---|:---|:---|
| 嵌套层级过多 | 可读性差 | 用逻辑与或多分支优化 |
| else匹配歧义 | 逻辑错误 | 使用花括号明确 |
| 内层变量外部使用 | 编译错误 | 在外层定义变量 |
| 条件顺序错误 | 某些分支永不执行 | 从严格到宽松排列 |

4. **变量作用域**：内层变量只在内层有效
5. **else匹配**：与最近的未匹配if匹配
6. **嵌套优化**：用逻辑与或多分支减少嵌套层级
7. **实际应用**：登录验证、三角形判断等多维度判断
