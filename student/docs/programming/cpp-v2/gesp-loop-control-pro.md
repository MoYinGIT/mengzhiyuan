## 【大题1】break的"立即跳出陷阱"

**考察点**：break立即终止整个循环，不再执行循环体剩余代码和条件判断

```cpp
#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 5; i++) {
        cout << i << " ";
        if (i == 3) {
            break;
        }
        cout << "* ";
    }
    cout << "结束" << endl;
    return 0;
}
```

### 一、选择题

**1. 程序的输出结果是（ ）**

A. 1 * 2 * 3 * 结束  
B. 1 * 2 * 3 结束  
C. 1 2 3 结束  
D. 1 * 2 * 3 4 5 结束

**2. 当i等于3时，程序会（ ）**

A. 输出3 * 然后继续下一次循环  
B. 输出3 然后立即退出循环  
C. 跳过输出3 直接退出  
D. 什么都不做

**3. break执行后，i++还会执行吗（ ）**

A. 会  
B. 不会  
C. 看情况而定  
D. 编译错误

### 二、判断题

**4. break语句会立即终止当前所在的整个循环。**  
（ ）正确 （ ）错误

**5. break执行后，循环的更新表达式（如i++）不会再执行。**  
（ ）正确 （ ）错误

**6. break可以终止if语句。**  
（ ）正确 （ ）错误

### 三、填空题

**7. break的作用是______当前所在的循环。**

**8. 当i=3时，程序输出3后遇到______，直接跳到循环外。**

**9. break只终止______（最内层/所有层）循环。**

---

**解析关键点**：
- break立即终止整个循环
- 循环体内break后的代码不执行
- break只作用于最内层循环

### 三、填空题

**7. continue的作用是______本次循环剩余代码，进入______。**

**8. 当i=3时，输出3后遇到______，跳过了输出*。**

**9. break是______循环，continue是______本次循环。**

- continue跳过本次循环剩余代码

## 【大题3】break与continue的"while陷阱"

```cpp
    int i = 1;

        if (i == 3) {

        i++;
    }
    cout << "结束" << endl;
    return 0;
}
```

**1. 程序的输出结果是（ ）**

A. 1 2 4 5 结束  
B. 1 2 结束

D. continue不能用于while

**3. 如何修复程序使其正确输出1 2 4 5（ ）**

A. 把continue改成break  
B. 把i++移到if前面  
C. 在continue前添加i++  
D. 把while改成for

### 二、判断题

**4. 在while循环中使用continue要特别注意循环变量的更新。**  
（ ）正确 （ ）错误

**5. 如果把i++放在continue之前，程序就不会死循环。**  
（ ）正确 （ ）错误

**6. for循环中使用continue不会导致死循环，因为i++在for头中。**  
（ ）正确 （ ）错误

### 三、填空题

**7. 在while循环中，continue可能导致______。**

**8. 修复方法是在continue______（之前/之后）更新循环变量。**

**9. for循环的更新表达式在______中，所以continue不会跳过它。**

---

**解析关键点**：
- while中continue可能跳过循环变量更新
- for循环的i++在头部，continue后仍会执行
- 修复方法：把i++放在continue之前

**学习建议**：
1. **break**：立即退出整个循环，不再执行循环体剩余代码
2. **continue**：跳过本次循环剩余代码，直接进入下一次循环
3. **return**：退出整个函数，比break更彻底
4. **while+continue陷阱**：注意循环变量的更新位置
5. **嵌套循环**：break/continue只作用于最内层循环
6. **多层退出**：使用标志变量配合break
7. **死循环+break**：while(true)配合条件break的经典模式
