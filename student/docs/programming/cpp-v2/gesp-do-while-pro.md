## 【大题1】do-while的"至少执行一次"特性

**考察点**：do-while循环先执行后判断，至少执行一次

```cpp
#include <iostream>
using namespace std;

int main() {
    int i = 10;
    do {
        cout << i << " ";
        i++;
    } while (i < 5);
    cout << endl;
    return 0;
}
```

### 一、选择题

**1. 程序的输出结果是（ ）**

A. （空，什么都不输出）  
B. 10  
C. 10 11 12 13 14  
D. 编译错误

**2. 如果将do-while改成while，程序的输出结果是（ ）**

A. 10  
B. 10 11 12 13 14  
C. （空，什么都不输出）  
D. 编译错误

**3. 以下关于do-while和while的区别，正确的是（ ）**

A. 两者完全相同，可以互换使用  
B. do-while先判断后执行，while先执行后判断  
C. do-while先执行后判断，while先判断后执行  
D. do-while只能用于数字循环

### 二、判断题

**4. do-while循环的条件表达式后面必须加分号。**  
（ ）正确 （ ）错误

**5. do-while循环的循环体至少会执行一次。**  
（ ）正确 （ ）错误

**6. 当循环条件一开始就不满足时，do-while和while的执行结果相同。**  
（ ）正确 （ ）错误

### 三、填空题

**7. do-while循环的执行顺序是：先______，后______。**

**8. 如果希望循环体至少执行一次，应该使用______循环。**

**9. 当i=10且条件为i<5时，do-while会输出______，while会输出______。**

---

**解析关键点**：
- do-while先执行后判断
- 循环体至少执行一次
- while条件不满足时可能一次也不执行

## 【大题2】do-while的"忘记分号陷阱"

```cpp
using namespace std;

int main() {

    return 0;

```

**1. 程序编译的结果是（ ）**

A. 正常编译，输出 1 2 3 结束  
B. 正常编译，输出 1 2 3 4 5 ...无限循环  
C. 编译错误  
D. 运行时崩溃

**2. 导致编译错误的原因是（ ）**

A. i没有初始化  
B. while条件缺少分号  
C. do后面不能有大括号  
D. i++位置错误

**3. 在第9行末尾添加什么可以修复错误（ ）**

A. 什么都不加，代码正确  
B. 添加分号;  
C. 添加逗号,  
D. 添加冒号:

### 二、判断题

**4. do-while的条件表达式后面必须加分号，这是语法要求。**  
（ ）正确 （ ）错误

**5. while循环的条件后面也需要加分号。**  
（ ）正确 （ ）错误

**6. 如果把while条件后面的分号去掉，程序可能会把下一行代码当作while语句的一部分。**  
（ ）正确 （ ）错误

### 三、填空题

**7. do-while的正确格式是：`do { ... } while (条件)______`**

**8. 与while不同，do-while的分号表示______结束。**

**9. 忘记写分号会导致编译器把后面的代码当作______的一部分。**

---

**解析关键点**：
- do-while条件后必须有分号
- while条件后不能加分号（加分号会变成空循环体）
- 分号是do-while语句的结束标志

D. 直接结束程序

**2. 如果用while替代do-while，需要如何修改（ ）**

A. 完全不需要修改  
B. 需要先写一次输入代码，再用while循环  
C. 把do和while的位置互换  
D. while不能用于输入验证

**3. 为什么输入验证场景适合用do-while而不是while（ ）**

A. do-while运行速度更快

**4. 输入验证是do-while循环的典型应用场景。**  
（ ）正确 （ ）错误

**5. 用while实现同样的输入验证功能，需要重复写一次输入代码。**  
（ ）正确 （ ）错误

**6. do-while的循环条件为真时继续循环，为假时退出循环。**  
（ ）正确 （ ）错误

### 三、填空题

**7. 输入验证场景使用do-while的好处是：______至少执行一次。**

**8. 如果用while实现输入验证，需要在循环______先执行一次输入。**

**9. do-while的循环条件`score < 0 || score > 100`表示______。**

**解析关键点**：
- 输入验证是do-while的经典应用场景

- while实现需要重复写输入代码

（ ）正确 （ ）错误

### 三、填空题

**7. 死循环的原因是循环体内缺少______语句。**

## 【大题10】do-while的"综合应用陷阱"

**考察点**：综合运用do-while解决实际问题

```cpp
#include <iostream>

int main() {
    int choice;
    int sum = 0;
    do {
        cout << "1. 加法 2. 显示结果 0. 退出：";
        cin >> choice;
        if (choice == 1) {
            int x;
            cout << "请输入数字：";
            cin >> x;
            sum += x;
        } else if (choice == 2) {
            cout << "当前和：" << sum << endl;
        }
    } while (choice != 0);
    cout << "最终和：" << sum << endl;
    return 0;
}
```

B. 只能加法计算  
C. 只能显示结果  
D. 无限循环

**2. 用户输入1后输入5，sum的值变为（ ）**

A. 0  
B. 1  
C. 5  
D. 6

**3. 如果用户输入1->3->1->4->2->0，输出结果是（ ）**

A. 当前和：0 最终和：0  
B. 当前和：7 最终和：7  
C. 当前和：3 最终和：4  
D. 当前和：7 最终和：0

### 二、判断题

**4. 这是一个简单的菜单系统，使用do-while确保菜单至少显示一次。**  
（ ）正确 （ ）错误

**5. 输入0会立即退出循环，不会执行后面的输出语句。**  
（ ）正确 （ ）错误

**6. 输入2不会修改sum的值，只是显示当前值。**  
（ ）正确 （ ）错误

### 三、填空题

**7. 菜单系统使用do-while的原因是：菜单______至少显示一次。**

**8. 条件`choice != 0`表示用户______输入0时继续循环。**

**9. 这是一个______累加器，每次输入的数字都会被加到sum中。**

**解析关键点**：
